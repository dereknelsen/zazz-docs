The dialog uses the native HTML `<dialog>` element, opened and closed entirely through the **Invoker Commands API** — no JavaScript. The [reset](app/zazz/styles/_reset.css) wires up spring-eased open and close transitions and a `--shade-800` backdrop. The browser handles the focus trap and `Esc`-to-close for free.

Use a dialog for confirmations, forms, and any modal interaction where the rest of the page should pause.

## Anatomy

```html
<button class="button" command="show-modal" commandfor="dialog-1">Open dialog</button>

<dialog id="dialog-1" class="dialog" closedby="any">
  <div class="dialog__content">
    <header class="dialog__header">
      <h2 class="text-lg font-heading weight-strong">Dialog title</h2>
      <p class="text-muted-foreground">Optional supporting copy.</p>
    </header>

    <div class="dialog__body prose">
      <!-- Forms, content, anything -->
    </div>
  </div>

  <footer class="dialog__footer">
    <button class="button" commandfor="dialog-1" command="close">Cancel</button>
    <button class="button" data-variant="primary" commandfor="dialog-1" command="close">
      Accept
    </button>
  </footer>

  <button
    class="dialog__close button absolute top-xs right-xs"
    data-variant="ghost"
    data-size="icon"
    commandfor="dialog-1"
    command="close"
    aria-label="Close dialog"
  >
    <svg><!-- x icon --></svg>
  </button>
</dialog>
```

| Part          | Class / attribute           | Notes                                                                                                                          |
| ------------- | --------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| Root          | `dialog.dialog`             | The native `<dialog>` element. Needs an `id` for the invoker to target.                                                        |
| Content       | `dialog__content`           | The scrolling region (`max-block-size: 100svh −` gaps). Wraps header + body.                                                   |
| Header        | `dialog__header`            | Title + supporting copy. Padded with `--gap-sm`.                                                                               |
| Body          | `dialog__body`              | Main content area, padded `--gap-sm` inline / `--gap-md` block. Pair with `prose` for long-form copy.                     |
| Footer        | `dialog__footer`            | Sticky bottom action bar with `border-top` and `--gap-xs` between actions. Sits outside `dialog__content`.                     |
| Footer action | `button` (+ `data-variant`) | A footer action is a plain `button` with `command="close"` and `commandfor`. The primary action adds `data-variant="primary"`. |
| Close button  | `dialog__close`             | Absolutely-positioned icon button (`button` with `data-variant="ghost"` `data-size="icon"`). Add `aria-label`.                 |

## Open and close via Invoker Commands

The native Invoker Commands API wires open and close with no JavaScript:

- A `<button command="show-modal" commandfor="dialog-1">` opens the dialog whose `id` matches `commandfor`.
- A `<button command="close" commandfor="dialog-1">` closes it. Place these in the footer and as the top-right close icon.
- `closedby="any"` on the `<dialog>` enables backdrop-click (light) dismissal.

The browser handles the backdrop, focus trap, and `Esc`-to-close. The trigger does not need to be near the dialog in the DOM — it targets by `id`, so any number of triggers anywhere on the page can open the same dialog.

```html
<button class="button" command="show-modal" commandfor="dialog-1">Open</button>
<!-- elsewhere on the page -->
<dialog id="dialog-1" class="dialog" closedby="any"><!-- … --></dialog>
```

## Open and close manually

When the invoker pattern doesn't fit (programmatic flows, cross-frame links), call the native API directly:

```js
const dialog = document.querySelector(".dialog");
dialog.showModal(); // opens with backdrop + focus trap
dialog.close();
```

The browser handles the backdrop, focus trap, and `Esc`-to-close. Zazz's reset adds the visual transitions on top.

## Sizes

Set `data-size` on the `.dialog` to change its width:

| `data-size`   | Width                                                      |
| ------------- | ---------------------------------------------------------- |
| none          | `min(50ch, 100% - var(--gap-md) * 2)` — character-based   |
| `article`     | `--article` (≈40rem)                                       |
| `container`   | `--container` (≈80rem)                                     |
| `screen`      | Full viewport (`100vi × 100svh`), no radius, no shadow     |

```html
<dialog class="dialog" data-size="article" closedby="any"><!-- … --></dialog>
<dialog class="dialog" data-size="container" closedby="any"><!-- … --></dialog>
<dialog class="dialog" data-size="screen" closedby="any"><!-- … --></dialog>
```

## What the reset adds

- Spring-eased opacity + scale transitions (`0.98` → `1`) on open and close, via `@starting-style` and the `--default-transition` easing/duration.
- A backdrop that animates from `--shade-none` to `--shade-900`. It's a **shade** deliberately, so the backdrop dims the page in **both** light and dark modes (`--muted` would lighten the page in dark mode).
- Scrollbar gutter handling while a dialog is open.

## Tokens

| Property        | Token / value                                                             |
| --------------- | ------------------------------------------------------------------------- |
| Surface         | `--card` background, `--card-foreground` text                             |
| Radius          | `--radius-lg` (≈16px) — applied directly                                  |
| Elevation       | `--shadow-md`                                                             |
| Overflow        | `clip`                                                                    |
| Header padding  | `--gap-sm`                                                                |
| Body padding    | `--gap-sm` inline, `--gap-md` block                                       |
| Footer          | `border-top: 1px solid var(--border)`, `--gap-xs` between actions, sticky |
| Backdrop        | `--shade-none` → `--shade-900` (fades in)                                 |
| Open transition | `--default-transition` easing + duration (set in reset)                   |
| Default width   | `min(50ch, 100% - var(--gap-md) * 2)`                                     |

## Accessibility

- Use the native `<dialog>` element. It handles the focus trap, `Esc` to close, and the inert background.
- `aria-label` or `aria-labelledby` on the dialog root is required when the title isn't immediately discoverable. Pointing `aria-labelledby` at the `dialog__header` heading's `id` is the typical pattern.
- The close button needs an `aria-label` (e.g., "Close dialog") since it's icon-only.
- The first focusable element receives focus automatically when the dialog opens. Order your markup so the right element gets focus (often the first input or the close button).
