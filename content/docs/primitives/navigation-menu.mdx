---
title: Navigation menu
description: Horizontal site navigation with rich popover panels and mega-menus. Popover API plus CSS anchor positioning, no JavaScript.
---

A navigation menu is the horizontal bar of top-level destinations, where some items open a panel of grouped links — a simple list, a two-column "featured" layout, or a full mega-menu. It's the dropdown's bigger sibling: same **Popover API** + **CSS anchor positioning** foundation, scaled up to richer panels, and still **no JavaScript**. The trigger chevron even rotates on open through a `:has()` selector.

## Default

<ComponentPreview name="navigation-menu-default" paddingClass="p-8" align="start" />

```html
<nav class="navigation-menu">
  <menu class="navigation-menu__container">
    <li class="navigation-menu__item">
      <button class="button navigation-menu__trigger" data-variant="ghost" popovertarget="nav-products">
        Products
        <svg viewBox="0 0 256 256" aria-hidden="true"><!-- chevron --></svg>
      </button>

      <div id="nav-products" class="navigation-menu__popover" popover="auto">
        <div class="navigation-menu__viewport grid grid-cols-2 gap-sm">
          <a href="#" class="navigation-menu__featured flex flex-col gap-xs p-md bg-muted">
            <span class="text-eyebrow text-muted-foreground">Featured</span>
            <span class="text-md weight-strong">Zazz Platform</span>
            <span class="text-sm text-muted-foreground text-pretty">Design, build, and ship from one token set.</span>
          </a>
          <div class="grid">
            <a href="#" class="navigation-menu__link">
              <span class="text-sm weight-strong">Analytics</span>
              <span class="text-sm text-muted-foreground">Real-time product insights.</span>
            </a>
            <a href="#" class="navigation-menu__link">
              <span class="text-sm weight-strong">Reports</span>
              <span class="text-sm text-muted-foreground">Share what the data says.</span>
            </a>
          </div>
        </div>
      </div>
    </li>

    <!-- a plain top-level link needs no popover -->
    <li class="navigation-menu__item">
      <a href="#" class="button" data-variant="ghost">Pricing</a>
    </li>
  </menu>
</nav>
```

## Anatomy

Each `<li>` is either a plain link or a trigger-plus-panel pair. The trigger is a ghost [`.button`](/docs/primitives/button); the panel is a `[popover]` anchored to that trigger. Panel content lives in a `.navigation-menu__viewport` grid.

| Part         | Element                                          | Role                                                       |
| ------------ | ------------------------------------------------ | ---------------------------------------------------------- |
| Root         | `<nav class="navigation-menu">`                  | Scopes the root anchor for full-width panels               |
| List         | `<menu class="navigation-menu__container">`      | The horizontal row of items                                |
| Item         | `<li class="navigation-menu__item">`             | Scopes a per-trigger anchor                                |
| Trigger      | `<button class="navigation-menu__trigger">`      | Ghost button; opens the panel; chevron rotates on open     |
| Panel        | `<div class="navigation-menu__popover">`         | `popover="auto"`; positioned against the trigger           |
| Viewport     | `<div class="navigation-menu__viewport">`        | Grid that lays out the panel content                       |
| Link         | `<a class="navigation-menu__link">`              | A panel link row (title + description)                     |
| Featured     | `<a class="navigation-menu__featured">`          | A tall callout cell for a two-column panel                 |
| Submenu      | `<div class="navigation-menu__submenu">` + `__submenu-trigger` | A nested flyout inside a panel                |

## Variants

Set these on the `.navigation-menu__popover`:

| Attribute       | Values                          | Effect                                                                                  |
| --------------- | ------------------------------- | --------------------------------------------------------------------------------------- |
| `data-align`    | `start` (default) · `center` · `end` | Inline alignment of the panel under its trigger. Panels open downward only.         |
| `data-size`     | `container` · `root` · `screen` | Widen the panel to the container width, the nav root width, or the viewport — via `anchor-size()`. Use for mega-menus. |
| `data-variant`  | `submenu`                       | A nested flyout that anchors to its sub-trigger and opens to the inline-end, with `position-try-fallbacks` for edge flips. |
| `data-animation`| `slide-down`                    | Reveal the viewport with an accordion-style height animation (pairs with `data-size="screen"`). |

The panel **layout** (single-column list vs. two-column featured) is utility-driven on the `.navigation-menu__viewport` (`grid grid-cols-2`, etc.) plus an optional `.navigation-menu__featured` callout — it isn't a `data-*` flag.

## Tokens

| Token              | Controls                                   |
| ------------------ | ------------------------------------------ |
| `--card` / `--card-foreground` | Panel surface and text         |
| `--muted` / `--muted-foreground` | Featured callout and link descriptions |
| `--radius-md` / `--radius-sm` / `--radius-xs` | Panel and nested button radii |
| `--shadow-md`      | Panel elevation                            |
| `--gap-md` / `--gap-sm` | Panel padding and grid gaps           |
| `--screen-xs`      | Default panel min-width                     |

Buttons rendered inside the panel drop to `--radius-xs`. The open/close animation is the shared spring-eased popover transition from `_reset.css`.

## States

- **Closed.** Only the trigger row shows; the chevron points down.
- **Open.** The panel animates in; `.navigation-menu__item:has(.navigation-menu__popover:popover-open) … > svg` rotates the chevron `180deg`.
- **Hover.** Links and submenu triggers lift to a `--muted` background.
- **Light-dismiss.** Click outside or press `Esc`. Submenus are nested popovers, so opening one keeps its ancestors open.
- **Focus.** Triggers and links show the standard `--ring` outline on `:focus-visible`.

> **Browser support.** The Popover API is Baseline; **CSS anchor positioning** ships in Chromium 125+ but not yet Firefox or Safari. All anchored layout is gated behind `@supports`, so where it's missing the panel (mega-menu included) falls back to the UA-centered popover — it still opens, light-dismisses, and animates. See [Tabs → browser support](/docs/primitives/tabs) for the shared pattern.

## Accessibility

- Triggers are real `<button>`s wired with `popovertarget`; plain destinations are `<a>` — keyboard operation and `Esc`-to-close come from the platform.
- Give icon-only triggers an `aria-label`; keep decorative chevrons `aria-hidden="true"`.
- Use `<menu>` / `<li>` for the bar so it's announced as a list. For a full `role="menubar"` widget with roving focus, layer the menu roles on top in an app — the visual layer doesn't depend on them.
- Within a featured panel, lead each link with a strong title so screen-reader users get the label before the description.
