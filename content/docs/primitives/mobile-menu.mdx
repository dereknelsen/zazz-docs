**A mobile navigation that feels native — because it is.**

Zazz Mobile Menu is a full-screen overlay built on `<dialog>` + native `<details>` accordion. It composes `.dialog[data-size="screen"]` for the takeover and `.accordion` for nested expand/collapse sections. No JavaScript — Invoker Commands opens it, native dialog closes it, and `<details>` handles the expand/collapse.

Use it for responsive navigation on mobile/tablet viewports, typically hidden behind a hamburger trigger and shown as the mobile counterpart to a desktop navigation menu.

## Anatomy

```html
<button class="button" data-size="icon" type="button" command="show-modal" commandfor="mobile-menu-1" aria-label="Open menu">
  <svg><!-- hamburger icon --></svg>
</button>

<dialog id="mobile-menu-1" class="dialog mobile-menu" data-size="screen" data-animation="slide-right" closedby="any">
  <div class="mobile-menu__viewport">
    <div class="mobile-menu__header">
      <span class="text-md weight-strong">Menu</span>
      <button class="button" data-variant="ghost" data-size="icon" type="button" commandfor="mobile-menu-1" command="close" aria-label="Close menu">
        <svg><!-- X icon --></svg>
      </button>
    </div>

    <div class="mobile-menu__body">
      <div class="accordion">
        <details>
          <summary class="button" data-variant="ghost">
            Section title
            <svg><!-- chevron --></svg>
          </summary>
          <div class="flex flex-col pb-sm">
            <a href="#" class="button flex flex-col items-start py-xs" data-variant="ghost">
              <span class="text-sm weight-strong">Link title</span>
              <span class="text-sm text-muted-foreground">Description text.</span>
            </a>
          </div>
        </details>
      </div>
    </div>

    <div class="mobile-menu__footer">
      <button class="button w-full" data-variant="ghost" type="button">Sign in</button>
      <a href="#" class="button w-full" data-variant="primary">Get started</a>
    </div>
  </div>
</dialog>
```

| Part                    | Selector                      | Notes                                                              |
| ----------------------- | ----------------------------- | ------------------------------------------------------------------ |
| Trigger                 | External `<button>`           | Hamburger icon button. Uses Invoker Commands to open.              |
| Dialog                  | `.dialog.mobile-menu`         | Full-screen takeover via `data-size="screen"`.                     |
| Viewport                | `.mobile-menu__viewport`      | Grid layout: header / body / footer. Full height.                  |
| Header                  | `.mobile-menu__header`        | Sticky top bar with brand/title + close button.                    |
| Body                    | `.mobile-menu__body`          | Scrollable content area. Holds the accordion nav.                  |
| Footer                  | `.mobile-menu__footer`        | Sticky bottom bar for CTAs (sign in, get started).                 |
| Accordion sections      | `.accordion` inside body      | Native `<details>` for expand/collapse.                            |

## Nesting

Accordion sections can nest for subcategories — a `<details>` inside a `<details>`:

```html
<details>
  <summary class="button" data-variant="ghost">Products</summary>
  <div class="flex flex-col pb-sm">
    <div class="accordion">
      <details>
        <summary class="button" data-variant="ghost">Design</summary>
        <div class="flex flex-col pb-sm">
          <a href="#" class="button flex flex-col items-start py-xs" data-variant="ghost">
            <span class="text-sm weight-strong">Figma Kit</span>
            <span class="text-sm text-muted-foreground">Variables, styles, and components.</span>
          </a>
        </div>
      </details>
    </div>
  </div>
</details>
```

Nested accordions get `padding-inline-start: var(--gap-sm)` automatically to show hierarchy.

## Animations

Set `data-animation` on the dialog to control entry:

| Value           | Behavior                                                |
| --------------- | ------------------------------------------------------- |
| `slide-right`   | Slides in from the left edge (`translateX(-100%)`)      |
| (none/default) | Standard dialog fade + scale from `_dialog.css`         |

The slide-right animation overrides the dialog's default transition with a 500ms duration and disables scale/opacity transforms.

## Tokens consumed

| Token                          | Use                                          |
| ------------------------------ | -------------------------------------------- |
| `--mobile-menu-background`     | Local prop → `--background`                  |
| `--mobile-menu-foreground`     | Local prop → `--foreground`                  |
| `--gap-xs`                     | Header/footer block padding                  |
| `--gap-sm`                     | Body padding, nested indent, footer gap      |
| `--border`                     | Header bottom / footer top borders           |

## States

| State                   | Behavior                                              |
| ----------------------- | ----------------------------------------------------- |
| Closed                  | Hidden. Hamburger trigger visible.                    |
| Open                    | Full-screen takeover. Body scrolls independently.     |
| Section expanded        | `<details open>` — chevron rotates, content reveals.  |
| Multiple sections open  | Supported — independent expand/collapse.              |

## Accessibility

- The trigger button has `aria-label="Open menu"`.
- The close button has `aria-label="Close menu"`.
- `closedby="any"` enables Escape and backdrop dismiss.
- Native `<dialog>` provides focus trapping.
- Accordion uses native `<details>`/`<summary>` — keyboard accessible by default.
- Footer CTAs are full-width for easy touch targets.

## Platform features used

- **Invoker Commands API** — `command="show-modal"` / `command="close"`.
- **Native `<dialog>`** — focus trapping, Escape, backdrop.
- **Native `<details>`/`<summary>`** — expand/collapse without JavaScript.
- **`data-size="screen"`** — dialog variant for full viewport takeover.
