## Quick reference

| Class             | Properties                     |
| ----------------- | ------------------------------ |
| `m-xs` … `m-xl`   | `margin: var(--gap-*);`        |
| `mx-xs` … `mx-xl` | `margin-inline: var(--gap-*);` |
| `mx-auto`         | `margin-inline: auto;`         |
| `my-xs` … `my-xl` | `margin-block: var(--gap-*);`  |
| `my-auto`         | `margin-block: auto;`          |
| `mt-xs` … `mt-xl` | `margin-top: var(--gap-*);`    |
| `mt-auto`         | `margin-top: auto;`            |
| `mr-xs` … `mr-xl` | `margin-right: var(--gap-*);`  |
| `mr-auto`         | `margin-right: auto;`          |
| `mb-xs` … `mb-xl` | `margin-bottom: var(--gap-*);` |
| `mb-auto`         | `margin-bottom: auto;`         |
| `ml-xs` … `ml-xl` | `margin-left: var(--gap-*);`   |
| `ml-auto`         | `margin-left: auto;`           |

## Examples

### All sides

```html
<div class="m-md">Margin on all sides at var(--gap-md).</div>
```

### Inline / block axes

`mx-{size}` for horizontal margin, `my-{size}` for vertical. Logical properties; they flip in RTL.

```html
<div class="mx-lg my-md">Generous horizontal margin, modest vertical.</div>
```

### Centering with `auto`

The `-auto` variants set the relevant margin to `auto`. Most common pattern: `mx-auto` to horizontally center a fixed-width block.

```html
<div class="mx-auto" style="width: var(--container)">Centered horizontally.</div>
```

`mt-auto` is useful inside a flex column to push an element to the bottom; `ml-auto` does the same horizontally.

```html
<div class="flex">
  <span>Left-aligned</span>
  <span class="ml-auto">Pushed right</span>
</div>
```

### A single side

```html
<div class="mt-lg mb-sm">Lots of room above, a little below.</div>
```

## Spacing scale

Same scale as [Padding](/docs/reference/padding). The five named [semantic gap tokens](/docs/foundations/spacing).

| Token      | Value            | Pixels at 16px root |
| ---------- | ---------------- | ------------------- |
| `--gap-xs` | `var(--step-2)`  | 8px                 |
| `--gap-sm` | `var(--step-4)`  | 16px                |
| `--gap-md` | `var(--step-6)`  | 24px                |
| `--gap-lg` | `var(--step-11)` | 44px                |
| `--gap-xl` | `var(--step-24)` | 96px                |

## Negative margins

Zazz doesn't ship negative-margin utilities by default. If a layout calls for negative margin, apply it directly in CSS:

```css
.bleed-edge {
  margin-inline: calc(var(--gap-md) * -1);
}
```

Most cases that look like they need negative margins are better solved by adjusting padding on the container or by using `gap` on a flex/grid parent.

## Using a custom value

For sizes off the semantic scale, reach for [step tokens](/docs/foundations/spacing#the-step-scale) directly:

```css
.detail-label {
  margin-bottom: var(--step-3); /* 0.75rem */
}
```

## Customizing the scale

The margin utilities consume the same `--gap-*` tokens as padding and gap. Overriding a gap token retunes margin, padding, and gap together. See [Padding → Customizing the scale](/docs/reference/padding#customizing-the-scale).
