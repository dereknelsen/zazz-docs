**Three effect groups: shadows for elevation, states for focus, and motion for transitions.**

Effects are the visual signal that something has depth or that a user is interacting with it. Zazz keeps the surface small: six shadow levels, four focus-ring tokens, and a shared motion bundle. Effects are easy to overspend and hard to recover from. A page covered in shadows reads as cluttered. A page with one focus indicator reads as confident.

## Shadows

Six shadow tokens, layered for soft falloff.

```css
none: 0px 0px 0px 0px oklch(from var(--black) l c h / 0);

--shadow-xs:
  0 8px 2px 0 oklch(from var(--black) l c h / 0), 0 5px 2px 0 oklch(from var(--black) l c h / 0.01),
  0 3px 2px 0 oklch(from var(--black) l c h / 0.03),
  0 1px 1px 0 oklch(from var(--black) l c h / 0.04), 0 0 1px 0 oklch(from var(--black) l c h / 0.05);

--shadow-sm:
  0 25px 7px 0 oklch(from var(--black) l c h / 0),
  0 16px 6px 0 oklch(from var(--black) l c h / 0.01),
  0 9px 5px 0 oklch(from var(--black) l c h / 0.03),
  0 4px 4px 0 oklch(from var(--black) l c h / 0.04),
  0 1px 2px 0 oklch(from var(--black) l c h / 0.05);

--shadow-md:
  0 66px 18px 0 oklch(from var(--black) l c h / 0),
  0 42px 17px 0 oklch(from var(--black) l c h / 0.01),
  0 24px 14px 0 oklch(from var(--black) l c h / 0.03),
  0 11px 11px 0 oklch(from var(--black) l c h / 0.04),
  0 3px 6px 0 oklch(from var(--black) l c h / 0.05);

--shadow-lg:
  0 247px 69px 0 oklch(from var(--black) l c h / 0),
  0 158px 63px 0 oklch(from var(--black) l c h / 0.01),
  0 89px 53px 0 oklch(from var(--black) l c h / 0.03),
  0 39px 39px 0 oklch(from var(--black) l c h / 0.05),
  0 10px 22px 0 oklch(from var(--black) l c h / 0.06);

--shadow-xl:
  0 370px 104px 0 oklch(from var(--black) l c h / 0),
  0 237px 95px 0 oklch(from var(--black) l c h / 0.01),
  0 133px 80px 0 oklch(from var(--black) l c h / 0.04),
  0 59px 59px 0 oklch(from var(--black) l c h / 0.06),
  0 15px 33px 0 oklch(from var(--black) l c h / 0.07);
```

Each shadow stacks multiple entries with decreasing opacity and increasing vertical offset and blur. A single shadow with a large blur reads as a halo around an object. A stack of progressively softer shadows reads as natural elevation. The opacity values run small (`0.01` to `0.07` per layer) so the cumulative effect stays subtle.

```css
.card {
  box-shadow: var(--shadow-md);
}

.modal {
  box-shadow: var(--shadow-xl);
}
```

The colors derive from `oklch(from var(--black) l c h / α)`: pure black at varying opacities. Because the alpha is what's varying, the shadow color picks up the active background through transparency, so the same shadow reads correctly on light and dark surfaces without a second token.

Apply a shadow directly with a utility: `shadow-none`, `shadow-xs`, `shadow-sm`, `shadow-md`, `shadow-lg`, `shadow-xl`.

Levels by intent:

- **`none`.** Explicit "no shadow," for components that toggle elevation on and off.
- **`--shadow-xs`.** Borders that need a slight lift.
- **`--shadow-sm`.** Resting buttons, hover affordances.
- **`--shadow-md`.** Cards, dropdowns, popovers.
- **`--shadow-lg`.** Modals, side panels.
- **`--shadow-xl`.** The most elevated surface in a view (a single command palette, a centered dialog over a backdrop).

  <Accordion title="How do I add a custom shadow?">
    Define it like the shipped ones. Stack multiple `box-shadow` entries for layered depth, or use a single entry for a flat shadow:

    ```css
    :root {
      --shadow-popover:
        0 4px 6px -1px oklch(from var(--black) l c h / 0.1),
        0 2px 4px -2px oklch(from var(--black) l c h / 0.1);
    }
    ```

    Apply with `box-shadow: var(--shadow-popover)`.

    The shipped shadows stack multiple entries to get a softer, more natural falloff. A single shadow with a large blur reads as a halo around the element; a stack of small offset shadows reads as elevation. When defining your own, layer for softness if the surface is meant to feel elevated, single-entry if it's meant to feel pressed-on. Derive the color from `oklch(from var(--black) l c h / α)` so it stays in the same color space as the shipped shadows.

## Focus ring

The focus ring is an **outline**, composed from four tokens.

```css
--ring: var(--primary);
--ring-style: solid;
--ring-size: 2px;
--ring-offset: 1px;
```

`--ring` is the ring color (the active brand color). `--ring-style` and `--ring-size` set the outline's style and width, and `--ring-offset` becomes the `outline-offset`, the gap that separates the ring from any border the element already has. Because `--ring` references `--primary`, the ring adapts to mode automatically: primary lightens in dark mode.

The reset applies the ring to every focusable element on `:focus-visible`:

```css
:where(:focus-visible) {
  outline: var(--ring-size) var(--ring-style) var(--ring);
  outline-offset: var(--ring-offset);
}
```

Components that draw their own ring keep a transparent outline at rest, then flip only the color on focus. This way the outline reserves its space up front and the ring never shifts layout when it appears:

```css
.interactive {
  outline: var(--ring-size) var(--ring-style) oklch(from var(--ring) l c h / 0);
  outline-offset: var(--ring-offset);
}

.interactive:focus-visible {
  outline-color: var(--ring);
}
```

The `.ring` utility does the same thing for any element you want to opt in. Always use `:focus-visible`, not `:focus`. The ring should appear on keyboard navigation, not on mouse clicks. `:focus-visible` is the standard mechanism for that distinction.

## Motion

A single spring curve and duration drive transitions across the system, bundled into one `--default-transition`.

```css
--spring-easing: linear(0, 0.012 0.9%, 0.049 2%, …, 0.994 68.8%, 1);
--spring-duration: 0.333s;
```

`--spring-easing` is a `linear()` curve that approximates spring physics for smooth, natural motion. `--spring-duration` (`0.333s`) is the default duration. The two compose into a transition bundle that animates a broad set of properties at once:

```css
--default-transition-timing-function: var(--spring-easing, cubic-bezier(0.17, 0.84, 0.44, 1));
--default-transition-duration: var(--spring-duration, 0.333s);
--default-transition: /* color, background, transform, opacity, box-shadow, … */;
```

Both the timing function and duration carry a fallback: browsers that don't support `linear()` easing fall back to an `easeOutQuart` cubic-bezier, and the duration falls back to `0.333s`. Apply the bundle directly, or via the `.transition` utility:

```css
.menu {
  transition: var(--default-transition);
}
```
