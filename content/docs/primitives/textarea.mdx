---
title: Textarea
description: Multi-line text entry. Grows with content via field-sizing.
---

A textarea is a multi-line input for free-form copy: messages, descriptions, comments. Zazz's textarea styles the native `<textarea>` element to match the [Input](/docs/primitives/input) primitive and uses `field-sizing: content` so the field grows as the user types.

## Default

<ComponentPreview name="textarea-default" />

```html
<textarea class="textarea" placeholder="Your message…"></textarea>
```

## Tokens

| Property     | Token / value                                                  |
| ------------ | -------------------------------------------------------------- |
| Width        | `100%`                                                         |
| Min height   | `5lh`                                                          |
| Max height   | `12lh`                                                         |
| Padding      | `--field-padding` (`--step-2_5`)                               |
| Border       | `1px solid var(--field-border)` (default), `--primary` (focus) |
| Radius       | `--field-radius` (defaults to `--radius-md`)                   |
| Background   | `--field-background` (`--input`)                               |
| Color        | `--field-foreground` (`--input-foreground`)                    |
| Font         | `--font-size-sm`, line-height `--leading-sm`                   |
| Field sizing | `content` (auto-grows)                                         |
| Resize       | `vertical`                                                     |
| Focus ring   | `--ring` outline (`:focus-visible`)                            |

## Form composition

Wrap the textarea in a `.field` with a `.field__label`. Add a `.field__hint` for helper copy or a `.field__error` for validation, both sharing the field's bottom row.

```html
<div class="field">
  <label class="field__label" for="message">Message</label>
  <textarea class="textarea" id="message" placeholder="Your message…"></textarea>
</div>
```

## Auto-growing

`field-sizing: content` makes the textarea grow vertically as the user types. It clamps between `5lh` (its starting height, so an empty textarea has visible breathing room) and `12lh`, then scrolls. `resize: vertical` lets the user drag it taller within that range.

To disable auto-grow, override the property:

```css
.textarea-fixed {
  field-sizing: auto;
  resize: vertical;
}
```

## States

- **Default.** Border `--border`, background `--input`.
- **Hover.** Border lifts to `--field-border--hover`.
- **Focus.** Border switches to `--primary`; the focus ring is a `:focus-visible` **outline** (`outline-color: var(--ring)`), which follows the field's rounded corners.
- **Disabled.** Set the `disabled` attribute. The shared disabled state applies `opacity: 0.5` and `cursor: not-allowed`.
- **Invalid.** Validation hangs off `:user-invalid`: the border, ring, and label turn `--destructive` and the `.field__error` reveals only after the user commits (blur or submit), never while typing.

## Accessibility

- Every textarea needs a label, visible or via [`sr-only`](/docs/reference/screen-readers).
- `placeholder` is for examples or hints, not labels.
- For long-form composition (essays, code), consider adding `aria-describedby` to a character count or formatting note.
