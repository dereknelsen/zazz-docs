**A gallery that zooms to a fullscreen slideshow with a single attribute.**

Zazz Lightbox composes a responsive thumbnail grid with a fullscreen `<dialog>` containing an Embla Carousel for swiping between images. Opening and closing use the Invoker Commands API — no JavaScript for the modal behavior. The carousel itself auto-initializes via the Embla data-attribute system.

Use it for image galleries, portfolio grids, product photo sets, and any "click to zoom, swipe to browse" pattern.

## Anatomy

```html
<div class="lightbox grid grid-cols-3 gap-xs">
  <button class="lightbox__trigger" type="button" command="show-modal" commandfor="lightbox-1" data-embla-start="0">
    <img src="thumb-1.jpg" alt="Description" loading="lazy" />
  </button>
  <button class="lightbox__trigger" type="button" command="show-modal" commandfor="lightbox-1" data-embla-start="1">
    <img src="thumb-2.jpg" alt="Description" loading="lazy" />
  </button>

  <dialog id="lightbox-1" class="lightbox__dialog dialog" data-size="screen" closedby="any">
    <div class="dialog__content">
      <div data-embla="root" data-embla-loop="true">
        <div data-embla="viewport">
          <div data-embla="container">
            <div class="lightbox__slide" data-embla="slide">
              <img src="full-1.jpg" alt="Description" />
            </div>
            <div class="lightbox__slide" data-embla="slide">
              <img src="full-2.jpg" alt="Description" />
            </div>
          </div>
        </div>
        <button class="lightbox__prev button" data-size="icon" data-variant="ghost" data-embla="prev" type="button" aria-label="Previous image">
          <svg><!-- left chevron --></svg>
        </button>
        <button class="lightbox__next button" data-size="icon" data-variant="ghost" data-embla="next" type="button" aria-label="Next image">
          <svg><!-- right chevron --></svg>
        </button>
      </div>
    </div>
    <button class="lightbox__close button" data-size="icon" data-variant="ghost" type="button" commandfor="lightbox-1" command="close" aria-label="Close lightbox">
      <svg><!-- X icon --></svg>
    </button>
  </dialog>
</div>
```

| Part                  | Selector                | Notes                                                                 |
| --------------------- | ----------------------- | --------------------------------------------------------------------- |
| Gallery grid          | `.lightbox`             | Base display grid. Use utility classes for columns and gap.           |
| Trigger               | `.lightbox__trigger`    | Button wrapping a thumbnail. Opens the dialog via Invoker Commands.   |
| Dialog                | `.lightbox__dialog`     | Fullscreen dark overlay. Extends `.dialog[data-size="screen"]`.       |
| Slide                 | `.lightbox__slide`      | Flexbox center for one full-size image. Also an Embla slide.          |
| Prev / Next           | `.lightbox__prev/next`  | Absolutely positioned nav buttons. Wired to Embla via `data-embla`.   |
| Close                 | `.lightbox__close`      | Top-right close button. Uses `command="close"`.                       |
| Counter (optional)    | `.lightbox__counter`    | Centered bottom indicator showing current/total.                      |

## Configuration

### Gallery columns and gap

Use grid utility classes on `.lightbox` to control columns and gap:

```html
<div class="lightbox grid grid-cols-2 gap-xs">…</div>
<div class="lightbox grid grid-cols-4 gap-sm">…</div>
<div class="lightbox grid grid-cols-5 gap-xs">…</div>
```

The component no longer uses `data-columns` — standard grid utilities (`grid-cols-*`, `gap-*`) give you full control.

### Starting slide

Each trigger's `data-embla-start` attribute tells the carousel which slide to jump to when opened:

```html
<button class="lightbox__trigger" command="show-modal" commandfor="lb-1" data-embla-start="2">
  <!-- Opens to slide index 2 (third image) -->
</button>
```

### Carousel options

The Embla root accepts standard carousel data attributes:

```html
<div data-embla="root" data-embla-loop="true">…</div>
```

## Tokens consumed

| Token                        | Use                                          |
| ---------------------------- | -------------------------------------------- |
| `--lightbox-radius`          | Thumbnail corner radius (`--radius-md`)      |
| `--lightbox-slide-radius`    | Fullscreen image radius (`--radius-lg`)      |
| `--gap-md`                   | Nav button offset from edges, slide padding  |
| `--gap-sm`                   | Close button offset from edges               |
| `--tint-100`                 | Subtle border on fullscreen images           |
| `--default-transition-*`     | Hover scale animation on thumbnails          |
| `--ring-color` / `--ring-opacity` | Focus-visible ring via `color-mix`      |

## States

| State          | Behavior                                                      |
| -------------- | ------------------------------------------------------------- |
| Trigger hover  | Thumbnail scales up slightly (`scale: 1.03`)                  |
| Focus-visible  | Standard `--ring` outline on the trigger button               |
| Dialog open    | Fullscreen dark backdrop; images centered; carousel active    |
| Nav hover      | White icon buttons at full opacity                            |

## Accessibility

- Each trigger is a `<button>` with `cursor: zoom-in` for affordance.
- Thumbnails carry `alt` text; fullscreen images repeat it.
- Nav and close buttons have `aria-label` attributes.
- `closedby="any"` allows backdrop click and Escape to dismiss.
- `loading="lazy"` on thumbnails for performance.

## Platform features used

- **Invoker Commands API** — `command="show-modal"` / `command="close"` for dialog open/close.
- **Native `<dialog>`** — focus trapping, backdrop, Escape handling.
- **Embla Carousel** — swipe, loop, prev/next, auto-init via data attributes.
