## Class reference

| Class       | CSS                                                                                 |
| ----------- | ----------------------------------------------------------------------------------- |
| `container` | `width: var(--container); margin-inline: auto; container: container / inline-size;` |
| `article`   | `width: var(--article); margin-inline: auto; container: article / inline-size;`     |

Both classes set an automatic max-width with built-in side gutter via `min()` math, center horizontally, and establish an inline-size container query context.

## Examples

```html
<section>
  <div class="container">
    <h2 class="text-h2">Section heading</h2>
    <p class="text-md">Body copy capped at --container width.</p>
  </div>
</section>

<article class="article">
  <h1 class="text-h1">Long-form heading</h1>
  <p class="text-md">Body capped at --article width for comfortable line length.</p>
</article>
```

You don't need a wrapping `.container` div with the class on it. Apply the class to the section, article, or div that holds the content.

## Tokens

```css
:root {
  --article: min(var(--screen-xs), 100% - var(--gap-md) * 2); /* 40rem max, 24px gutter */
  --container: min(var(--screen-lg), 100% - var(--gap-md) * 2); /* 80rem max, 24px gutter */
}
```

What the `min()` does:

- **On wide viewports** the element resolves to the named screen width (`--screen-xs` or `--screen-lg`).
- **On narrow viewports** it resolves to `100% - (gutter × 2)`. The 24px side padding is built in.

## Container queries

The `container: <name> / inline-size` declaration registers each as a container query context. Child elements can respond to the container's width.

### Breakpoint utilities

Zazz ships prefixed utilities that apply when the nearest inline-size container is **wider than** a `--screen-*` token:

| Prefix | Token        | Threshold |
| ------ | ------------ | --------- |
| `xs:`  | `--screen-xs` | 40rem    |
| `sm:`  | `--screen-sm` | 48rem    |
| `md:`  | `--screen-md` | 64rem    |
| `lg:`  | `--screen-lg` | 80rem    |
| `xl:`  | `--screen-xl` | 96rem    |

```html
<section class="container">
  <div class="flex flex-col gap-md sm:flex-row">
    <aside class="hidden sm:block">Shows when container &gt; 48rem</aside>
    <main class="col-span-12 md:col-span-8">…</main>
  </div>
</section>
```

See [Display](/docs/reference/display), [Flexbox](/docs/reference/flexbox), and [Grid](/docs/reference/grid) for the full class lists (`{bp}:hidden`, `{bp}:flex-row`, `{bp}:col-span-*`, etc.).

### Custom `@container` rules

```css
/* Custom rules may use var(--screen-*) where your bundler allows it */
@container (width > 48rem) {
  .feature {
    grid-template-columns: 1fr 1fr;
  }
}

@container article (width > 40rem) {
  .quote {
    font-size: var(--font-size-h4);
  }
}
```

Unnamed queries match any inline-size ancestor (including `section` from the reset). Named queries (`container`, `article`) scope to that wrapper only.

## Adding container sizes

The `min()` pattern is composable. Define new container variables with different max widths:

```css
:root {
  --container-narrow: min(var(--screen-md), 100% - var(--gap-md) * 2);
  --container-wide: min(var(--screen-xl), 100% - var(--gap-md) * 2);
}

.container-narrow {
  width: var(--container-narrow);
  margin-inline: auto;
}
```

See [Foundations → Layout](/docs/foundations/layout) for the architectural choice and rationale.
