---
title: Accordion
description: Stacked disclosure panels built on native <details>. Animated open state, zero JavaScript.
---

An accordion is a stack of headers that each reveal a panel of content. Reach for one when you have several short sections that don't all need to be visible at once — FAQs, settings groups, supporting detail under a heading. Zazz builds the accordion from native `<details>` / `<summary>` elements, so the toggle, keyboard support, and open state come from the browser with **no JavaScript**. The open/close height animates through the `::details-content` pseudo-element and `interpolate-size: allow-keywords`.

## Default

```html
<div class="accordion">
  <details open>
    <summary>
      What is an accordion?
      <svg viewBox="0 0 256 256" aria-hidden="true">
        <polyline points="208 96 128 176 48 96" fill="none" stroke="currentColor" stroke-width="16" />
      </svg>
    </summary>
    <div class="pb-sm">
      <p>A disclosure widget built from native <code>&lt;details&gt;</code> and <code>&lt;summary&gt;</code>.</p>
    </div>
  </details>

  <details>
    <summary>
      How does it work?
      <svg viewBox="0 0 256 256" aria-hidden="true">
        <polyline points="208 96 128 176 48 96" fill="none" stroke="currentColor" stroke-width="16" />
      </svg>
    </summary>
    <div class="pb-sm">
      <p>Each item is a separate <code>&lt;details&gt;</code>, so several can stay open at once.</p>
    </div>
  </details>
</div>
```

## Anatomy

The `.accordion` is just a container. Each row is a native `<details>` whose `<summary>` is the clickable header and whose remaining content is the panel. Items are separated by a bottom border (every `details` but the last), and the chevron `<svg>` inside the summary rotates `-180deg` when the item is open.

| Part      | Element                      | Role                                                          |
| --------- | ---------------------------- | ------------------------------------------------------------ |
| Root      | `<div class="accordion">`    | Grid container; draws the dividers between items             |
| Item      | `<details>`                  | One disclosure row; carries its own `[open]` state           |
| Header    | `<summary>`                  | The clickable trigger — flex row with text and chevron       |
| Indicator | `<svg>` inside `<summary>`   | Chevron; rotates `-180deg` on `[open]` (decorative)          |
| Panel     | `<div class="pb-sm">`        | Content wrapper; the `pb-sm` utility spaces it from the next |

Each item is independent, so **multiple panels can be open at the same time**. For a single-open ("exclusive") accordion, give every `<details>` the same `name` attribute — the browser then closes the others automatically.

## Tokens

The accordion consumes theme tokens directly rather than exposing a large local-prop set:

| Token                  | Controls                                              |
| ---------------------- | ---------------------------------------------------- |
| `--border`             | The 1px divider between items                         |
| `--gap-sm`             | `summary` block padding                              |
| `--weight-strong`      | `summary` text weight                                |
| `--step-4`             | Chevron size                                         |
| `--default-transition` | Spring-eased open/close and chevron rotation         |
| `--decoration-*`       | The animated underline offset on the summary         |

## States

- **Resting.** The summary sits at `--weight-strong` with an underline; the panel is collapsed.
- **Hover.** The summary's `text-underline-offset` lifts (via `--decoration-offset--hover`).
- **Open.** `[open]` rotates the chevron `-180deg` and animates the panel height open through `::details-content`.
- **Focus.** The summary is natively focusable and shows the standard `--ring` outline on `:focus-visible`.

> **Browser support.** The height animation relies on `interpolate-size: allow-keywords` (set on `:root`) and transitions on `::details-content`. Where unsupported, the panel still opens and closes — it just snaps instead of sliding. The disclosure itself works everywhere `<details>` does (Baseline).

## Accessibility

- `<details>` / `<summary>` give you a button-like trigger with `Enter` / `Space` toggling and correct expanded/collapsed semantics for free — no `aria-expanded` wiring needed.
- Keep the chevron `<svg>` decorative with `aria-hidden="true"`; the summary text is the accessible name.
- Use a real heading inside the summary (or wrap the summary text in one) when the accordion sits in a content hierarchy, so the document outline stays meaningful.
- For an exclusive accordion, the shared `name` attribute is announced correctly by the platform — no extra ARIA.
