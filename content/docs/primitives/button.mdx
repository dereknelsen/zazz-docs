A button is the element users press to do a thing. Apply the `button` class to an `<a>`, `<button>`, or any clickable element. Add a `data-variant` attribute to choose the visual treatment.

## Variants

| Variant                  | Use for                                                            | shadcn equivalent            |
| ------------------------ | ------------------------------------------------------------------ | ---------------------------- |
| `button` (no attribute)  | Default outlined treatment. Neutral, fits anywhere.                | `button` (default / outline) |
| `data-variant="primary"` | The page's most important action. Filled with `--primary`.         | `button-primary` (default)   |
| `data-variant="muted"`   | A secondary action that still asks for attention. Soft background. | `button-secondary`           |
| `data-variant="ghost"`   | Tertiary actions and toolbars. Transparent until hover.            | `button-ghost`               |
| `data-variant="link"`    | Inline text-style action that doesn't need to look like a button.  | `button-link`                |

Variants are data attributes, not modifier classes. The base `button` class sets sizing, padding, and shared focus behavior; the `data-variant` re-points the local color props. The default (no attribute) is the outlined treatment.

<ComponentPreview name="button-variants" />

```html
<a class="button" href="/contact">Get in touch</a>
<a class="button" data-variant="primary" href="/signup">Start free trial</a>
<a class="button" data-variant="muted" href="/save">Save draft</a>
<a class="button" data-variant="ghost" href="/help">Help</a>
<a class="button" data-variant="link" href="/learn">Learn more</a>
```

## With icons

Drop an `<svg>` directly inside `.button` â€” there's no wrapper element. It auto-sizes to `--button-icon-size` (`--step-4`) via `:where(.button > svg)`. Put the icon before or after the bare text to set its position.

<ComponentPreview name="button-with-icons" />

```html
<a class="button" data-variant="primary" href="/signup">
  Start free trial
  <svg><!-- arrow icon --></svg>
</a>
<a class="button" href="/read">
  <svg><!-- arrow icon --></svg>
  Read more
</a>
```

## Icon-only

Set `data-size="icon"` for a square icon-only button at the default button height. Drop the text, put the `<svg>` directly inside, and add `aria-label` to the root. `data-size="icon-sm"` gives a smaller square, and `data-size="sm"` gives a compact (shorter) button that still holds text.

<ComponentPreview name="button-minimal" />

```html
<button class="button" data-size="icon" aria-label="Open menu">
  <svg><!-- menu icon --></svg>
</button>

<a class="button" data-variant="primary" data-size="icon" aria-label="Save" href="/save">
  <svg><!-- check icon --></svg>
</a>
```

## Anatomy

```html
<a class="button" href="/example">
  <svg><!-- optional leading icon --></svg>
  Label
  <svg><!-- optional trailing icon --></svg>
</a>
```

| Part    | Selector                                     | Notes                                                           |
| ------- | -------------------------------------------- | --------------------------------------------------------------- |
| Root    | `.button`                                    | The clickable element. Required.                                |
| Variant | `data-variant="primary\|muted\|ghost\|link"` | Optional. Omit for the default outlined treatment.              |
| Size    | `data-size="sm\|icon\|icon-sm"`              | Optional. `sm` compact, `icon` square, `icon-sm` small square.  |
| Icon    | `.button > svg`                              | A direct-child SVG. Auto-sized; use one on each side as needed. |
| Label   | bare text                                    | The visible label, no wrapper. Omit for icon-only buttons.      |
| Hint    | `.button > kbd`                              | Optional keyboard hint. The button reserves end padding for it. |

## Tokens

| Property                   | Token / value                                                                                                        |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| Height                     | `--button-height` (defaults to `--step-8`)                                                                           |
| Padding (inline)           | `--button-padding` (`--step-2_5`)                                                                                    |
| Gap between icon and label | `--button-gap` (`--step-1`)                                                                                          |
| Icon size                  | `--button-icon-size` (`--step-4`)                                                                                    |
| Border                     | `1px solid var(--border)` (default), variant-dependent elsewhere                                                     |
| Radius                     | `--button-radius` (defaults to `--radius-md`)                                                                        |
| Background (default)       | `--background`                                                                                                       |
| Background (primary)       | `--primary` â†’ text `--primary-foreground`                                                                            |
| Background (muted)         | `--muted`, no visible border â†’ text `--foreground`                                                                   |
| Background (ghost)         | transparent â†’ hover paints `--muted`                                                                                 |
| Background (link)          | no box, underlined â†’ hover sets text to `--primary`                                                                  |
| Font                       | family `--font-body`, size `--font-size-sm`, weight `--weight-strong`, line-height `1`                               |
| Hover                      | background shifts to `--muted` (default) or drops opacity (primary)                                                  |
| Active                     | background-color `--active` shade (primary/ghost) â€” see `_button.css`                                                |
| Focus-visible              | `outline-color: var(--ring)` (transparent outline held at rest; or the [`ring`](/docs/reference/focus-ring) utility) |

The button declares `--button-radius: var(--radius-md)` locally â€” there is no `--radius-button` token. Override `--button-radius` to retune just the button's corners, or shift `--radius-multiplier` (in [Foundations â†’ Radius](/docs/foundations/radius)) to move the whole system.

## Accessibility

- Use `<button>` for actions in the page and `<a class="button">` for navigation. The class works on both; the element conveys the semantic.
- For icon-only buttons (`data-size="icon"` with no text), set `aria-label` describing the action.
- Focus state uses `:focus-visible`, so keyboard users see the outline ring and mouse users don't.
- Disabled buttons should set the `disabled` attribute on `<button>` or `aria-disabled="true"` on `<a>`. The button drops to `0.5` opacity and ignores pointer events. `aria-busy="true"` shows a progress cursor.
- Interactive states (hover/active/focus) apply only to `button.button` and `a[href].button`.
