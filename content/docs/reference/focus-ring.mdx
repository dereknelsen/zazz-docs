## Class reference

| Class  | Property                                                                                                       |
| ------ | -------------------------------------------------------------------------------------------------------------- |
| `ring` | `outline: var(--ring-size) var(--ring-style) <transparent>` → `outline-color: var(--ring)` on `:focus-visible` |

The `ring` utility lays down a transparent outline at rest and only swaps in the ring color on `:focus-visible`, so keyboard navigation shows the ring without mouse clicks triggering it — and without shifting layout, because the outline always occupies the same space.

## Tokens

```css
--ring: var(--primary);
--ring-style: solid;
--ring-size: 2px;
--ring-offset: 1px;
```

Four tokens describe the ring:

- **`--ring`** — the ring color. Defaults to `--primary`, so it tracks the brand and adapts to mode automatically (primary lightens in dark mode).
- **`--ring-style`** — the outline style (`solid`).
- **`--ring-size`** — the outline width (`2px`).
- **`--ring-offset`** — the gap between the element's edge and the ring, via `outline-offset` (`1px`). This is what separates the ring from any border the element already has, so the two don't fight.

There is **no** `--focus-ring` box-shadow token; the focus ring is outline-based.

## Usage

The reset already applies the ring on every `:focus-visible`, so most elements need nothing:

```css
:where(:focus-visible) {
  outline: var(--ring-size) var(--ring-style) var(--ring);
  outline-offset: var(--ring-offset);
}
```

Components that want a non-shifting ring keep a transparent outline at rest and switch only the color on focus — the same pattern as the `ring` utility:

```css
.btn {
  outline: var(--ring-size) var(--ring-style) oklch(from var(--ring) l c h / 0);
  outline-offset: var(--ring-offset);
}

.btn:focus-visible {
  outline-color: var(--ring);
}
```

Or just add the utility class:

```css
.btn {
  @apply ring;
}
```

The Zazz primitives ([Button](/docs/primitives/button), [Badge](/docs/primitives/badge), [Input](/docs/primitives/input), and friends) all consume the same `:focus-visible` outline, so the four ring tokens tune the ring everywhere at once.

## `:focus` vs `:focus-visible`

Use `:focus-visible`, not `:focus`. The ring should appear on keyboard navigation, not on mouse clicks. `:focus-visible` is the browser's mechanism for that distinction.

```css
.btn:focus-visible {
  outline-color: var(--ring);
}
```

Modern browsers handle the distinction automatically when you target `:focus-visible`, so you don't need a secondary `:focus:not(:focus-visible)` rule to suppress the ring on clicks.

## Customizing

Retune the ring for the whole product by overriding the tokens:

```css
:root {
  --ring: var(--destructive); /* destructive ring for a delete flow */
  --ring-size: 3px;
}
```

For a high-contrast accessibility mode, thicken the ring and push it out:

```css
.high-contrast {
  --ring: var(--foreground);
  --ring-size: 4px;
  --ring-offset: 2px;
}
```

Components that consume the `:focus-visible` outline follow the override automatically.

## Why outline instead of box-shadow

Modern `outline` follows `border-radius` — a rounded button gets a rounded ring, not a rectangular one — which is exactly why Zazz uses an outline ring. `outline` also paints outside the box without being clipped by `overflow: hidden`, and `outline-offset` pushes the ring away from the element so it reads as a separate halo. That offset is what provides the separation from any existing border; there's no `--background`-colored gap layer to manage and no box-shadow to keep in sync.
