---
title: Backgrounds
description: Background-color utilities from the semantic role tokens, plus the shade/tint overlay scales and transparent helpers. Channel alpha and hover variants included.
---

`bg-*` utilities set `background-color` from the **theme role tokens**, so surfaces swap correctly
between light and dark. Like the text utilities, each value is
`oklch(from var(--role) l c h / calc(alpha × --_background-alpha))`, so
[color-opacity](/docs/base/utilities/color-opacity) can dim just the background channel.

Background color is the one place Zazz also ships utilities for a numeric scale, the **shade**
and **tint** overlays, because dimming or lightening a surface is so common.

## Surfaces & roles

| Class                                | Token                              |
| ------------------------------------ | ---------------------------------- |
| `bg-background` / `bg-foreground`    | `--background` / `--foreground`    |
| `bg-card` / `bg-card-foreground`     | `--card` / `--card-foreground`     |
| `bg-input` / `bg-input-foreground`   | `--input` / `--input-foreground`   |
| `bg-border` / `bg-border-foreground` | `--border` / `--border-foreground` |
| `bg-muted` / `bg-muted-foreground`   | `--muted` / `--muted-foreground`   |
| `bg-faded` / `bg-faded-foreground`   | `--faded` / `--faded-foreground`   |

## Brand & status

| Brand                                      | Status                                         |
| ------------------------------------------ | ---------------------------------------------- |
| `bg-primary` / `bg-primary-foreground`     | `bg-info` / `bg-info-foreground`               |
| `bg-secondary` / `bg-secondary-foreground` | `bg-success` / `bg-success-foreground`         |
| `bg-tertiary` / `bg-tertiary-foreground`   | `bg-warning` / `bg-warning-foreground`         |
|                                            | `bg-destructive` / `bg-destructive-foreground` |

## Absolute & empty

| Class            | Result                                    |
| ---------------- | ----------------------------------------- |
| `bg-white`       | `--white` (does not flip in dark mode)    |
| `bg-black`       | `--black` (does not flip in dark mode)    |
| `bg-transparent` | `background-color: transparent`           |
| `bg-none`        | `background: none` (clears image + color) |

## Overlay scales

Two alpha-based overlay ramps for dimming and lightening any surface. `*-none` is fully
transparent; `*-full` is the most opaque step.

| Scale              | Classes                                                        | Built from                       | Use                                           |
| ------------------ | -------------------------------------------------------------- | -------------------------------- | --------------------------------------------- |
| **Shade** (darken) | `bg-shade-none`, `bg-shade-50 … bg-shade-950`, `bg-shade-full` | `--shade-*` (from `neutral-950`) | scrims, modal backdrops (e.g. `bg-shade-800`) |
| **Tint** (lighten) | `bg-tint-none`, `bg-tint-50 … bg-tint-950`, `bg-tint-full`     | `--tint-*` (from white)          | frosted highlights over imagery               |

Because they're alpha overlays, they read the underlying surface through. Layer one over media or
a colored panel rather than painting a solid fill.

```html
<div class="grid grid-area-pile">
  <img src="…" alt="…" class="size-full object-cover" />
  <div class="bg-shade-700"></div>
  <h2 class="text-white self-end p-md">Readable over any photo</h2>
</div>
```

## Variants

- **Hover:** every role plus `transparent`/`none` has a `hover:bg-*` counterpart. See
  [Hover states](/docs/base/utilities/hover).
- **Opacity:** dim just the background channel with `bg-opacity-*`. See
  [Color opacity](/docs/base/utilities/color-opacity). It multiplies into the token's own alpha, so
  shade/tint/faded keep their built-in translucency.

<Callout type="info">
  The full numeric brand scales (`--primary-600`, `--neutral-100`, …) have **no** `bg-` utility
  classes by design. Reach a fixed shade through the token in custom CSS or inline. The role tokens
  above cover the everyday cases. See the [token
  reference](/docs/getting-started/overview#core-files) for the scale model.
</Callout>
