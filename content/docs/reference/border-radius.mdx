## Class reference

| Class          | Properties                                                       |
| -------------- | ---------------------------------------------------------------- |
| `rounded-0`    | `border-radius: 0;`                                              |
| `rounded-xs`   | `border-radius: var(--radius-xs);   /* step-1   × multiplier */` |
| `rounded-sm`   | `border-radius: var(--radius-sm);   /* step-1.5 × multiplier */` |
| `rounded-md`   | `border-radius: var(--radius-md);   /* step-2.5 × multiplier */` |
| `rounded-lg`   | `border-radius: var(--radius-lg);   /* step-4   × multiplier */` |
| `rounded-xl`   | `border-radius: var(--radius-xl);   /* step-7   × multiplier */` |
| `rounded-full` | `border-radius: var(--radius-full); /* 9999rem, capsule */`      |

## Examples

```html
<div class="rounded-md p-md" style="background: var(--card)">A card with medium radius.</div>

<button
  class="rounded-full px-lg py-sm"
  style="background: var(--primary); color: var(--primary-foreground)"
>
  Pill button
</button>
```

## Tokens

```css
:root {
  /* modifier */
  --radius-multiplier: 1;

  /* semantic */
  --radius-0: 0rem;
  --radius-xs: calc(var(--step-1) * var(--radius-multiplier));
  --radius-sm: calc(var(--step-1_5) * var(--radius-multiplier));
  --radius-md: calc(var(--step-2_5) * var(--radius-multiplier));
  --radius-lg: calc(var(--step-4) * var(--radius-multiplier));
  --radius-xl: calc(var(--step-7) * var(--radius-multiplier));
  --radius-full: 9999rem;
}
```

There are **no** primitive radius aliases (`--radius-button`, `--radius-input`, `--radius-badge`, `--radius-card` no longer exist). Each component reaches the semantic scale through a **local** custom property instead — `_button.css` declares `--button-radius: var(--radius-md)`, `_badge.css` declares `--badge-radius: var(--radius-sm)`, the form controls share `--field-radius: var(--radius-md)`, and `_dialog.css` uses `--radius-lg` directly.

## Customizing

Adjust `--radius-multiplier` to sharpen or round every radius globally (it doesn't affect `--radius-none` or `--radius-full`):

```css
.architectural {
  --radius-multiplier: 0;
} /* sharp */
.friendly {
  --radius-multiplier: 1.5;
} /* rounder */
```

Retune a single component by overriding its local radius prop, or point it at a different step on the semantic scale:

```css
.button {
  --button-radius: var(--radius-full); /* every button a capsule */
}
```

See [Foundations → Radius](/docs/foundations/radius) for the nested-radius math and the local-prop design rationale.
