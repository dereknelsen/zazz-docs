---
title: Radio
description: Mutually exclusive selection within a group. Styled native HTML control.
---

A radio is one of a set of mutually exclusive options. Picking one deselects the others in the same group. Use radios for binary or small fixed-option choices where the visible options matter. For longer lists, reach for [Select](/docs/primitives/select) instead.

## Default

<ComponentPreview name="radio-group" />

```html
<fieldset class="radio-group">
  <legend>Plan</legend>
  <label class="field">
    <input class="radio" type="radio" name="plan" value="starter" />
    <span class="field__label">Starter</span>
  </label>
  <label class="field">
    <input class="radio" type="radio" name="plan" value="pro" />
    <span class="field__label">Pro</span>
  </label>
</fieldset>
```

`.radio-group` is a `<fieldset>` that shares layout with `.field-group` (see `_fields.css`). Each option is a `<label class="field" >` pairing a `.radio` input with a `.field__label`. Radios in the same group share a `name` attribute, and the browser handles the mutual exclusion.

## Tokens

| Property       | Token / value                                                |
| -------------- | ------------------------------------------------------------ |
| Width / height | `--step-4_5`                                                 |
| Border         | `1px solid var(--border)` (unchecked), `--primary` (checked) |
| Radius         | `--radius-full`                                              |
| Background     | `--input` (unchecked), `--primary` (checked)                 |
| Checked dot    | `radial-gradient` in `--primary-foreground`                  |
| Inline gap     | `--step-2` (via `.field[]`)                                  |
| Group gap      | `--step-1_5` (via `.radio-group`)                            |
| Focus ring     | `--ring` outline (`:focus-visible`)                          |

The checked appearance fills the circle with `--primary` and paints the inner dot with a `radial-gradient` in `--primary-foreground`, so the dot tracks the theme rather than relying on color alone.

## States

- **Unchecked.** Border `--border`, background `--input`.
- **Hover.** Border lifts to `--primary`.
- **Checked.** Border and background switch to `--primary`; a `--primary-foreground` dot appears via `radial-gradient`.
- **Focus.** The focus ring is a `:focus-visible` outline (`outline-color: var(--ring)`).
- **Disabled.** Set `disabled` on the input. The shared disabled state applies `opacity: 0.5` and `cursor: not-allowed`.

## Composing a group

```html
<fieldset class="radio-group">
  <legend>Plan</legend>
  <label class="field">
    <input class="radio" type="radio" name="plan" value="starter" />
    <span class="field__label">Starter</span>
  </label>
  <label class="field">
    <input class="radio" type="radio" name="plan" value="pro" />
    <span class="field__label">Pro</span>
  </label>
  <label class="field">
    <input class="radio" type="radio" name="plan" value="team" />
    <span class="field__label">Team</span>
  </label>
</fieldset>
```

The `<fieldset class="radio-group">` with an optional `<legend>` gives the group an accessible name. The shared `name="plan"` keeps the radios mutually exclusive.

## Accessibility

- Wrap each radio in a `<label class="field" >` so clicking the label selects the option.
- Group related radios in a `<fieldset class="radio-group">` with a `<legend>` describing what the choice is for.
- Sharing a `name` attribute is what makes radios mutually exclusive. The browser also handles keyboard navigation (`Arrow` to move between options, `Space` to select).
- For a required radio group, use `required` on at least one input in the group.
