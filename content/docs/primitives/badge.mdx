---
title: Badge
description: Compact label for status, category, or count. Same variant set as button.
---

A badge is a small label attached to a piece of content. Use one when you need to call attention to category, status, count, or a tag value without the weight of a button.

Badges share the button's variant vocabulary so the two compose visually. A primary badge on a primary button reads as a coherent system rather than two separate decisions. The badge is a gently-rounded chip (`--badge-radius` defaults to `--radius-sm`), not a capsule.

## Variants

| Variant                  | Use for                                       | shadcn equivalent         |
| ------------------------ | --------------------------------------------- | ------------------------- |
| `badge` (no attribute)   | Default outlined treatment.                   | `badge` (outline)         |
| `data-variant="primary"` | "New," "Beta," "Live." Calls attention.       | `badge-primary` (default) |
| `data-variant="muted"`   | "Draft," "Archived." Soft, present-but-quiet. | `badge-secondary`         |
| `data-variant="ghost"`   | Inline metadata where a chip is too loud.     | `badge-ghost`             |
| `data-variant="link"`    | A badge that's also a link (`<a>` element).   | n/a                       |

Like the button, variants are `data-variant` attributes, not modifier classes. A `data-variant="link"` badge drops to `--badge-radius: 0` and underlines its text.

<ComponentPreview name="badge-variants" />

```html
<span class="badge">Category</span>
<span class="badge" data-variant="primary">New</span>
<span class="badge" data-variant="muted">Draft</span>
<span class="badge" data-variant="ghost">Inline</span>
<a class="badge" data-variant="link" href="/posts">Posts</a>
```

## Icon-only

Set `data-size="icon"` for a square icon-only badge. Useful for dots, status indicators, or kebab-menu triggers. Put the `<svg>` directly inside and add `aria-label` to the root. (The badge has only the `icon` size; the button additionally has `sm` and `icon-sm`.)

<ComponentPreview name="badge-minimal" />

```html
<span class="badge" data-size="icon" aria-label="More actions">
  <svg><!-- dots icon --></svg>
</span>
```

## Anatomy

```html
<span class="badge">
  <svg><!-- optional leading icon --></svg>
  Label
  <svg><!-- optional trailing icon --></svg>
</span>
```

| Part    | Selector                                     | Notes                                                  |
| ------- | -------------------------------------------- | ------------------------------------------------------ |
| Root    | `.badge`                                     | A `<span>`, `<a>`, or `<button>` element. Required.    |
| Variant | `data-variant="primary\|muted\|ghost\|link"` | Optional. Omit for the default outlined treatment.     |
| Size    | `data-size="icon"`                           | Optional. Square icon-only treatment.                  |
| Icon    | `.badge > svg`                               | A direct-child SVG. Auto-sized to `--badge-icon-size`. |
| Label   | bare text                                    | The visible text, no wrapper. Uses `--font-size-xs`.   |

## Tokens

| Property             | Token / value                                                                                                        |
| -------------------- | -------------------------------------------------------------------------------------------------------------------- |
| Height               | `--badge-height` (defaults to `--step-5`)                                                                            |
| Padding (inline)     | `--badge-padding` (`--step-2`)                                                                                       |
| Gap                  | `--badge-gap` (`--step-1`)                                                                                           |
| Icon size            | `--badge-icon-size` (`--step-3`)                                                                                     |
| Border               | `1px solid var(--border)` (default), variant-dependent elsewhere                                                     |
| Radius               | `--badge-radius` (defaults to `--radius-sm`, a gentle chip)                                                          |
| Background (default) | `--background`                                                                                                       |
| Background (primary) | `--primary` → text `--primary-foreground`                                                                            |
| Background (muted)   | `--muted`, no visible border → text `--foreground`                                                                   |
| Background (ghost)   | transparent → hover paints `--muted`                                                                                 |
| Background (link)    | no box, underlined → hover sets text to `--primary`                                                                  |
| Font                 | family `--font-body`, size `--font-size-xs`, weight `--weight-strong`, line-height `1`                               |
| Hover                | `--muted` background on default; opacity drop on primary                                                             |
| Active               | active shade (link also bumps `text-underline-offset`)                                                               |
| Focus-visible        | `outline-color: var(--ring)` (transparent outline held at rest; or the [`ring`](/docs/reference/focus-ring) utility) |

The badge declares `--badge-radius: var(--radius-sm)` locally — there is no `--radius-badge` token. Override `--badge-radius` to make square chips (`--radius-none`), capsules (`--radius-full`), or matched-to-button corners (`--radius-md`). See [Foundations → Radius](/docs/foundations/radius).

## States

Interactive states (hover, active, focus-visible) apply **only** to `button.badge` and `a[href].badge`. A bare `<span class="badge">` is inert decoration. Clickable badges (root is `<a href>` or `<button>`) inherit the same hover, active, and focus-visible behavior as `button`.

## Accessibility

- Static badges (decorative labels) don't need ARIA. Use a plain `<span>` — it carries no interactive state.
- Clickable badges should be `<a>` (navigation) or `<button>` (action). Use `data-variant="link"` for the text-style treatment.
- A badge that conveys live status (e.g., "Live now") should sit inside a region with `aria-live="polite"` if it changes dynamically.
- Color alone shouldn't carry meaning. Pair color with an icon or label for success/warning/error badges.
- For `data-size="icon"`, set `aria-label` on the root.
