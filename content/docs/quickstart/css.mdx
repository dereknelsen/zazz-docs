Zazz is a set of small CSS partials. **Variables** hold the token surface, **reset** sets sensible element defaults, **typography** and **utilities** ship the Tailwind-style helper classes, and there's one partial per component. Cascade order is set once, in `index.css`, with a single `@layer` declaration. None of it requires JavaScript or a build step — any project that can import CSS can run Zazz.

## 1. Get the files

The files live at [app/zazz/styles/](app/zazz/styles/) in the Zazz repo:

- [index.css](app/zazz/styles/index.css): the entry file. Contains **only** the layer declaration — `@layer variables, reset, components, utilities;` — which sets the cascade order.
- [\_variables.css](app/zazz/styles/_variables.css): every Zazz token, organized into the collections → `@layer variables`
- [\_reset.css](app/zazz/styles/_reset.css): box-sizing, element defaults, the focus ring, native dialog/popover enhancements → `@layer reset`
- [\_typography.css](app/zazz/styles/_typography.css): the `text-*` and `.prose` type styles → `@layer utilities`
- [\_utilities.css](app/zazz/styles/_utilities.css): color, display, flex, grid, spacing, size, radius, shadow, and layout utility classes → `@layer utilities`
- one partial per component (`_badge.css`, `_button.css`, `_accordion.css`, `_dialog.css`, `_dropdown.css`, `_navigation-menu.css`, `_fields.css`, `_input.css`, `_textarea.css`, `_select.css`, `_input-group.css`, `_password-group.css`, `_radio.css`, `_tabs.css`) → `@layer components`. `_fields.css` carries the shared form glue (the `--field-*` tokens, `.field` wrapper, and validation).

Copy the styles folder into your project. A common location is `/styles/zazz/`.

<Accordions>
  <Accordion title="What's in each file?">
    `index.css` is one line: the `@layer variables, reset, components, utilities;` declaration. Because the order is fixed here, the other partials can be imported in any order — each wraps its own rules in its layer.

    `_variables.css` declares the token surface: theme (base, overlay, brand, status), corporate scales (primary, secondary, tertiary), grayscale (neutral, shade, tint), typography, spacing, radius, layout, effects, plus the dark-mode wiring.

    `_reset.css` sets `box-sizing: border-box` on everything, removes default margins and padding, normalizes form element styling, applies the outline-based focus ring on `:focus-visible`, and adds spring-eased transitions to native `<dialog>` and `[popover]` elements.

    `_typography.css` and `_utilities.css` ship Tailwind-style atomic classes for type, flex, grid, gap, padding, margin, radius, shadow, container, and ring. Every class consumes Zazz tokens, so changes to the variables propagate through the utilities automatically.

    The component partials (`_button.css`, `_badge.css`, …) each style one component in `@layer components`, which sits between reset and utilities — so utility classes always win over a component's baseline.

  </Accordion>
</Accordions>

## 2. Import

**With a bundler** (Vite, Next, Astro, SvelteKit, …), import the JS entry once in a shared layout and let the bundler handle the CSS:

```ts
import "./zazz/styles/load.ts";
```

`load.ts` imports `index.css` first (which declares the layer order) and then every partial. One line wires the whole system.

**Without a bundler**, make an entry stylesheet that imports `index.css` first — so the layers exist before any rules land in them — then the partials in any order:

```css
@import "./zazz/styles/index.css"; /* declares @layer variables, reset, components, utilities */

@import "./zazz/styles/_variables.css";
@import "./zazz/styles/_reset.css";
@import "./zazz/styles/_typography.css";
@import "./zazz/styles/_utilities.css";

@import "./zazz/styles/_badge.css";
@import "./zazz/styles/_button.css";
@import "./zazz/styles/_accordion.css";
@import "./zazz/styles/_dialog.css";
@import "./zazz/styles/_dropdown.css";
@import "./zazz/styles/_navigation-menu.css";
@import "./zazz/styles/_fields.css";
@import "./zazz/styles/_input.css";
@import "./zazz/styles/_textarea.css";
@import "./zazz/styles/_select.css";
@import "./zazz/styles/_input-group.css";
@import "./zazz/styles/_password-group.css";
@import "./zazz/styles/_radio.css";
@import "./zazz/styles/_tabs.css";
```

Import only the component partials you actually use — they're independent. One caveat for the form set: `.input`, `.textarea`, `.select`, `.input-group`, and `.password-group` share the `--field-*` tokens declared in `_fields.css`, so import `_fields.css` whenever you use any of them.

The layer order in `index.css` does the sorting, not import order: utility classes override the component baseline, which overrides the reset, which sits at the bottom. Your own un-layered CSS wins over all of them.

## 3. Verify

Drop a button on the page that uses Zazz tokens and utility classes:

```html
<button
  class="p-md rounded-md text-md"
  style="background: var(--primary); color: var(--primary-foreground);"
>
  Confirm
</button>
```

A rounded button in your primary brand color with `--gap-md` of padding. If the button is unstyled, your imports failed. If the colors are missing, check the path to `_variables.css`.

## Dark mode

Zazz responds to system preference (`prefers-color-scheme: dark`) and to a `.dark` class on any ancestor. Force a mode at the root:

```html
<html class="dark">
  ...
</html>
```

See [Theme: How modes work](/docs/foundations/theme#how-modes-work) for the full pattern, including how to add a third mode.
