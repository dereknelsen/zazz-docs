A carousel is an Embla slider wrapped in `<embla-carousel>`. The element is the carousel
root: put Embla options such as `data-embla-loop` and `data-embla-classnames` on it, then
compose the usual `viewport` → `container` → `slide` structure inside. Each slide sets its
own responsive width with `basis-*` utilities
(`basis-full xs:basis-1/2 sm:basis-1/3 md:basis-1/4`).

`<embla-carousel>` uses light DOM, so the markup you write is the markup that gets styled.
It initializes when connected, defers initialization while inside a closed `<dialog>`, and
destroys its Embla instances if the element is removed. The preview loads the required
scripts automatically; open the JS tab to inspect the custom element.

## API

Mark up the roles with `data-embla`, and put options on `<embla-carousel>`.

| Attribute                             | Where | Purpose                                         |
| ------------------------------------- | ----- | ----------------------------------------------- |
| `<embla-carousel>`                    | root  | Carousel custom element; holds all options      |
| `data-embla="viewport"`               | child | The visible window (required)                   |
| `data-embla="container"`              | child | The flex track of slides                        |
| `data-embla="slide"`                  | child | Each slide — set width with `basis-*` utilities |
| `data-embla="prev"` / `"next"`        | child | Navigation buttons                              |
| `data-embla-loop`                     | root  | `"true"` to wrap around                         |
| `data-embla-align`                    | root  | `start`, `center`, `end`                        |
| `data-embla-autoplay` / `-autoscroll` | root  | Plugins (with their `-*` options)               |

Legacy `data-embla="root"` wrappers still work for lower-level Embla examples. Prefer
`<embla-carousel>` for component examples and any markup that may be inserted or removed
dynamically.

The lightbox is built on the same Embla roles plus a fullscreen `<dialog>` and a thumb
strip — see [Lightbox](/docs/components/lightbox).
