Responsive utilities in Zazz are **container-query based**. A prefix like `md:grid-cols-3` applies
when the element's nearest **container** is wide enough — not the viewport. The same component
adapts correctly whether it's in a full-width section or a narrow sidebar, because it responds to
the space it's actually in.

## The prefixes

Five breakpoints, applied like Tailwind's: prefix a utility with `xs:`, `sm:`, `md:`, `lg:`, or
`xl:`. Each variant activates **at or above** its width (it's min-width-style), so the unprefixed
class is your base and prefixes layer on as the container grows.

| Prefix | Activates when container width is… | Token |
|---|---|---|
| *(none)* | always — the base | — |
| `xs:` | `> 40rem` | `--screen-xs` |
| `sm:` | `> 48rem` | `--screen-sm` |
| `md:` | `> 64rem` | `--screen-md` |
| `lg:` | `> 80rem` | `--screen-lg` |
| `xl:` | `> 96rem` | `--screen-xl` |

```html
<!-- 1 column by default, 2 from xs, 3 from sm, 4 from lg -->
<div class="grid grid-cols-1 xs:grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 gap-md">…</div>
```

<Callout type="info">
The thresholds are authored as literal `rem` values to match `--screen-*`, because `var()` isn't
allowed in a `@container` query prelude. The tokens and the breakpoints stay in lockstep, and you
can still use `--screen-*` inside `calc()` elsewhere.
</Callout>

## Which utilities are responsive

Responsive variants exist for the **layout** families — the ones you actually rearrange across
breakpoints. Each is available at every prefix (`xs:` through `xl:`):

| Family | Responsive classes |
|---|---|
| Display | `hidden` `block` `flex` `inline-flex` `grid` |
| Subgrid | `grid-cols-subgrid` `grid-rows-subgrid` |
| Grid tracks | `grid-cols-1…12` `grid-rows-1…12` |
| Grid spans | `col-span-1…12` `row-span-1…12` |
| Flex direction | `flex-row` `flex-col` |
| Flex basis | `basis-0` `basis-auto` `basis-full` `basis-1/2…1/6` |
| Align items | `items-start` `items-center` `items-end` `items-stretch` |
| Justify content | `justify-start` `justify-center` `justify-end` `justify-stretch` `justify-around` `justify-between` |
| Text align | `text-left` `text-center` `text-right` `text-justify` |

### What is base-only (no prefix)

To keep the stylesheet lean, less-toggled families ship **only** as base classes — apply them
outside the responsive layer:

- `self-*`, `justify-self-*`, `place-items-*`
- `visible` / `invisible`, and the `inline` / `inline-block` display values
- spacing (`p-*`, `m-*`, `gap-*`), sizing (`w-*`, `h-*`, `size-*`), and negative margins
- color, border, radius, shadow, opacity, and scale

If you need one of these to change at a breakpoint, drive it from a parent that *does* flip (e.g.
restructure with `md:flex-row`) rather than reaching for a per-breakpoint spacing class.

## Setting up a container

Responsive prefixes query the nearest ancestor that's a **container context**. Two things create
one:

- The page **body** is already a container (set in the reset), so utilities respond to the page
  width out of the box.
- The `.container` and `.article` wrappers each register their own context
  (`container: … / inline-size`). Nest content inside one and its responsive utilities measure
  *that* wrapper — which is what makes a component reusable in any column width.

```html
<section class="container">
  <!-- these respond to .container's width, not the window -->
  <div class="flex flex-col md:flex-row gap-md">…</div>
</section>
```

## Ordering

Source order in `_utilities.css` puts the base classes **before** the `@container` blocks, so a
prefixed variant always wins over its base when its breakpoint is active. Write them in natural
reading order — base first, then ascending prefixes:

```html
<div class="flex-col sm:flex-row lg:items-center">…</div>
```

## Container queries vs media queries

Because these are container queries, a card laid out with `grid-cols-1 md:grid-cols-2` becomes two
columns whenever **its container** passes `md` — in a wide main column it splits, in a narrow
sidebar it stays single, *on the same page, at the same viewport width*. That's the advantage over
viewport media queries: components stay correct wherever you drop them. Hit **Expand** on the
previews above to widen their container and watch the breakpoints fire.
