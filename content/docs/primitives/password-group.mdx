---
title: Password group
description: A masked password field with a show/hide reveal toggle, built on the shared form surface.
---

A password group is a [`.input`](/docs/primitives/input)`[type="password"]` wrapped in a bordered shell with one extra part: a show/hide reveal toggle. It inherits the shared `--field-*` surface, hover/focus, and `:user-invalid` validation from the rest of the form family, so it reads as one of the set. The icon swap is pure CSS off the toggle's `aria-pressed` state; flipping the input `type` is the one small piece of JavaScript any form control here asks for — and the field is a fully working masked input without it.

## Default

<ComponentPreview name="password-group-default" />

```html
<div class="field">
  <label class="field__label" for="password">Password</label>

  <label class="password-group">
    <input
      class="input"
      id="password"
      name="password"
      type="password"
      autocomplete="new-password"
      minlength="8"
      required
      placeholder="••••••••"
      aria-describedby="password-hint password-warning"
    />
    <span class="password-group__addon" data-align="inline-end">
      <button
        class="button password-group__toggle"
        type="button"
        data-variant="ghost"
        data-size="icon-sm"
        aria-pressed="false"
        aria-label="Show password"
        aria-describedby="password-warning"
      >
        <svg class="password-group__icon password-group__icon--show" viewBox="0 0 256 256" aria-hidden="true"><!-- eye --></svg>
        <svg class="password-group__icon password-group__icon--hide" viewBox="0 0 256 256" aria-hidden="true"><!-- eye-slash --></svg>
      </button>
    </span>
  </label>

  <div class="field__description">
    <span class="field__hint" id="password-hint">Use eight or more characters.</span>
    <span class="field__error" role="alert">Password must be at least 8 characters.</span>
  </div>
  <span class="sr-only" id="password-warning">Warning: showing the password makes it visible to anyone near your screen.</span>
</div>
```

## Reveal toggle script

The CSS swaps the eye / eye-slash icons from `aria-pressed`. A few lines wire the actual reveal — flip the input `type`, the toggle's `aria-pressed`, and its `aria-label`:

```js
const toggle = document.querySelector(".password-group__toggle");
const input = toggle.closest(".password-group").querySelector(".input");

toggle.addEventListener("click", () => {
  const show = toggle.getAttribute("aria-pressed") === "true";
  toggle.setAttribute("aria-pressed", String(!show));
  toggle.setAttribute("aria-label", show ? "Show password" : "Hide password");
  input.type = show ? "password" : "text";
});
```

## Anatomy

The shell is a `<label>` (so clicking anywhere focuses the input); the toggle lives in a trailing addon. Wrap the whole thing in a [`.field`](/docs/primitives/input#the-field-wrapper) for the label, hint, and error.

| Part      | Element                                  | Role                                                          |
| --------- | ---------------------------------------- | ------------------------------------------------------------ |
| Shell     | `<label class="password-group">`         | Bordered field shell; `:focus-within` lifts the whole shell  |
| Control   | `<input class="input" type="password">`  | The masked field; goes borderless inside the shell           |
| Addon     | `<span class="password-group__addon">`   | Trailing slot (`data-align="inline-end"`) holding the toggle |
| Toggle    | `<button class="password-group__toggle">`| Ghost `icon-sm` button; carries `aria-pressed`               |
| Icons     | `.password-group__icon--show` / `--hide` | The two SVGs; CSS shows one based on `aria-pressed`          |

## Tokens

The shell shares the form family's surface — override one local to retune the whole set:

| Token                   | Default              | Controls                          |
| ----------------------- | -------------------- | --------------------------------- |
| `--field-background`    | `--input`            | Shell fill                        |
| `--field-foreground`    | `--input-foreground` | Text color                        |
| `--field-border`        | `--border`           | Shell border                      |
| `--field-border--focus` | `--primary`          | Border on `:focus-within`         |
| `--field-radius`        | `--radius-md`        | Shell corner radius               |
| `--field-height`        | `--step-8`           | Shell height                      |
| `--ring`                | `--primary` (alpha)  | Focus outline on `:focus-visible` |

## States

- **Resting / hover / focus.** Same surface and border transitions as every field; `:focus-within` lifts the whole shell and shows the `--ring` outline.
- **Masked / revealed.** Driven by `aria-pressed` on the toggle: `false` shows the eye icon and a masked field; `true` shows the eye-slash icon and the script switches `type` to `text`.
- **Invalid.** On `:user-invalid` (after blur/submit, never while typing) the field flips border, ring, and label to `--destructive`, the hint hides, and the `.field__error` reveals.
- **Disabled.** Shared with the form family: `opacity: 0.5` + `cursor: not-allowed`.

## Accessibility

- Always pair the control with a real `.field__label[for]`.
- The toggle is keyboard-reachable, exposes `aria-pressed`, and swaps its `aria-label` between "Show password" and "Hide password" so its state is announced.
- An `sr-only` warning (referenced via `aria-describedby`) tells assistive-tech users that revealing exposes the password on screen.
- Keep the icons `aria-hidden="true"` — the button's label is the accessible name.
- Use `autocomplete="new-password"` on sign-up and never block pasting, so password managers can generate and fill.
