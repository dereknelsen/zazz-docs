Zazz is designed and maintained by **Derek Nelsen**.

It exists because most design-system work I've shipped for agencies ends up rebuilding the same eight token collections, three platforms at a time. The framework collects those reusable parts and ships them as a single coherent system — Figma, Webflow, Tailwind, and plain CSS speaking the same variable names.

## Built on the shoulders of

Zazz isn't a fork of any other system, but it's deeply influenced by what came before:

- **[Tailwind CSS](https://tailwindcss.com)** — for the variable-driven theme model and the 50–950 color scale convention.
- **[shadcn/ui](https://ui.shadcn.com)** — for the background-foreground theme naming and the philosophy of shipping code, not packages.
- **[Radix UI](https://www.radix-ui.com)** — for the unstyled primitive pattern and the discipline of accessible defaults.
- **[Tints.dev](https://www.tints.dev)** by Simeon Griggs — for the OKLCH scale generator that produces Zazz's default palettes.
- **[Linear](https://linear.app)** — for the proof that documentation can be both precise and warm.
