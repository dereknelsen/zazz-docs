title: Checkbox

A checkbox is the standard control for binary or multi-select choices: terms acceptance, feature toggles, filter inclusion. Zazz styles the native `<input type="checkbox">` directly in the reset (`appearance: none` redraws it from theme tokens) — so the same accessibility, keyboard handling, and form submission you get from the browser is wrapped in a Zazz-themed appearance. **No class is needed on the input.**

## Default

<ComponentPreview name="checkbox-default" />

```html
<label class="field">
  <input type="checkbox" />
  <span class="field__label">I agree to the terms</span>
</label>
```

Wrap the checkbox in a `<label class="field" >`. The ``attribute lays the control out in a row beside its`.field\_\_label`, with `--step-2`between them. The input itself carries no class; the reset styles every`input[type="checkbox"]` that isn't a switch.

## Tokens

| Property       | Token / value                                                |
| -------------- | ------------------------------------------------------------ |
| Width / height | `--step-4_5`                                                 |
| Border         | `1px solid var(--border)` (unchecked), `--primary` (checked) |
| Radius         | `--radius-xs`                                                |
| Background     | `--input` (unchecked), `--primary` (checked / indeterminate) |
| Inline gap     | `--step-2` (via `.field[]`)                                  |
| Focus ring     | `--ring` outline (`:focus-visible`)                          |

The checked and indeterminate states fill `--primary` and layer a glyph (a check or a dash) via an SVG mask — never color alone:

```css
input[type="checkbox"]:checked {
  background-color: var(--primary);
  border-color: var(--primary);
  background-image: var(--checkbox-checkmark-mask);
}
```

## States

- **Unchecked.** Border `--border`, background `--input`.
- **Checked.** Border + background switch to `--primary`; a checkmark SVG mask fills the box.
- **Indeterminate.** Set via JavaScript — `input.indeterminate = true` (there is no HTML attribute for it). The box fills `--primary` and shows a dash glyph.
- **Focus.** The focus ring is a `:focus-visible` outline (`outline-color: var(--ring)`).
- **Disabled.** Set `disabled` on the input. The shared disabled state applies `opacity: 0.5` and `cursor: not-allowed`.

## Accessibility

- The native `<input type="checkbox">` handles keyboard interaction (Space toggles), screen reader announcement, and form submission automatically.
- Wrap each checkbox in a `<label class="field" >` so clicking the label text toggles the input.
- For a group of related checkboxes (filter options, multi-select), wrap the group in `<fieldset>` with a `<legend>` describing the group.
- For required boolean inputs (terms acceptance), use `required` and `aria-describedby` if the label needs supplementary explanation.
