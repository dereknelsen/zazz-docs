title: Select
description: Customizable native select with a fully styled picker, options, and chevron.

A select is a dropdown that lets a user pick one option from a fixed list. Zazz styles the native `<select>` element with `appearance: base-select`, which themes the trigger, the dropdown panel, the options, and the chevron while the browser keeps keyboard navigation, top-layer rendering, and form integration. It matches the [Input](/docs/primitives/input) primitive's height, padding, radius, and theme.

## Default

<ComponentPreview name="select-default" />

```html
<select class="select">
  <button>
    <selectedcontent></selectedcontent>
  </button>
  <option value="" disabled selected>Select oneâ€¦</option>
  <option value="first">First choice</option>
  <option value="second">Second choice</option>
  <option value="third">Third choice</option>
</select>
```

The customizable select **requires** a `<button><selectedcontent></selectedcontent></button>` as its first child â€” the `<button>` is the trigger and `<selectedcontent>` mirrors the chosen option's markup. A `::picker-icon` chevron sits at the trailing edge and rotates when the picker is `:open`. Where `appearance: base-select` is unsupported, the control degrades to the native OS dropdown (the reset still supplies a chevron).

## Form composition

Wrap the select in a `.field` with a `.field__label`.

```html
<div class="field">
  <label class="field__label" for="plan">Plan</label>
  <select class="select" id="plan">
    <button>
      <selectedcontent></selectedcontent>
    </button>
    <option value="" disabled selected>Choose a plan</option>
    <option value="starter">Starter</option>
    <option value="pro">Pro</option>
    <option value="team">Team</option>
  </select>
</div>
```

## Tokens

| Property       | Token / value                                                     |
| -------------- | ----------------------------------------------------------------- |
| Width          | `100%`                                                            |
| Height         | `--field-height` (`--step-8`)                                     |
| Padding        | `--field-padding` inline-start, `--step-2` inline-end             |
| Border         | `1px solid var(--field-border)` (default), `--primary` (focus)    |
| Radius         | `--field-radius` (defaults to `--radius-md`)                      |
| Background     | `--field-background` (`--input`)                                  |
| Color          | `--field-foreground` (`--input-foreground`)                       |
| Font           | `--font-size-sm`                                                  |
| Picker panel   | `--card` background, `--border`, `--radius-md`, `--shadow-md`     |
| Picker icon    | `::picker-icon` chevron (`--field-icon-size`), rotates on `:open` |
| Checked option | `--primary-100` fill, `--primary` text, `--weight-strong`         |
| Focus ring     | `--ring` outline (`:focus-visible` and `:open`)                   |

## States

- **Default.** Border `--border`, background `--input`, chevron visible.
- **Hover.** Border lifts to `--field-border--hover`.
- **Open / focus.** Border switches to `--primary` and the focus ring (a `:focus-visible` outline) applies; the chevron rotates `-180deg` while the picker is open.
- **Checked option.** The selected `<option>` fills `--primary-100`, sets `--primary` text and `--weight-strong`, and shows a `::checkmark` â€” never color alone.
- **Disabled.** Set the `disabled` attribute on the select. The shared disabled state applies `opacity: 0.5` and `cursor: not-allowed`.
- **Required.** Use `required` on the select and a sentinel `<option value="" disabled selected>` to force a choice.

## Customizable vs. native

Zazz uses the native `<select>` with `appearance: base-select`, so the same element renders a fully themed picker â€” a card-like `::picker(select)` panel, styled `<option>` rows, a rotating `::picker-icon`, and a `::checkmark` on the selection â€” without any JavaScript. Keyboard navigation, screen reader announcement, top-layer rendering, and form submission stay native. Where the browser doesn't support `appearance: base-select`, the control gracefully degrades to the platform's native dropdown.

For richer dropdowns (multi-select, search, async loading), drop the `select` class and build a custom component. The customizable native select is the right primitive for simple, single-value choices.

## Accessibility

- Every select needs a label, visible or via [`sr-only`](/docs/reference/screen-readers).
- The placeholder option pattern (`<option value="" disabled selected>Chooseâ€¦</option>`) provides a default prompt without making it a selectable value.
- Native `<select>` handles keyboard navigation, screen reader announcement, and option focus automatically, even with the customizable picker.
- For grouped options, use `<optgroup>` with a `label` attribute.
