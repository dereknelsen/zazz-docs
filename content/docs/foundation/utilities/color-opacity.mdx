---
title: Color opacity
description: Dim a color channel (text, background, or border) without fading the element or its children, backed by non-inheriting @property alpha variables.
---

Color-opacity utilities dim **one color channel** and nothing else. Unlike
[`opacity-*`](/docs/base/utilities/effects#opacity), which fades the whole element (children included),
these multiply into a single token's alpha, so text stays crisp while a background softens, and a
child never inherits the dimming.

## The two opacity systems

|                     | What it affects                   | Classes                                                                |
| ------------------- | --------------------------------- | ---------------------------------------------------------------------- |
| **Element opacity** | the entire element + its children | `opacity-0 … opacity-100` ([Effects](/docs/base/utilities/effects#opacity)) |
| **Channel alpha**   | only the named color channel      | `text-opacity-*` · `bg-opacity-*` · `border-opacity-*`                 |

Both use the same quartile scale: `0`, `25`, `50`, `75`, `100`.

## Channel alpha utilities

Compose one with any matching color class: `bg-primary bg-opacity-50`, `text-foreground
text-opacity-75`, `border border-destructive border-opacity-25`.

| Channel    | Classes                                 | Sets                  |
| ---------- | --------------------------------------- | --------------------- |
| Text       | `text-opacity-0 … text-opacity-100`     | `--_text-alpha`       |
| Background | `bg-opacity-0 … bg-opacity-100`         | `--_background-alpha` |
| Border     | `border-opacity-0 … border-opacity-100` | `--_border-alpha`     |

```html
<span class="bg-primary bg-opacity-50 text-primary-foreground rounded-sm px-sm">50% surface</span>
<p class="text-foreground text-opacity-75">Slightly softened body text.</p>
```

## How it works: non-inheriting alpha variables

Every `text-*`, `bg-*`, and `border-*` color value is computed as a multiply, not a fixed alpha:

```css
:where(.bg-primary) {
  background-color: oklch(from var(--primary) l c h / calc(alpha * var(--_background-alpha, 1)));
}
```

`alpha` is the token's **own** alpha; `--_background-alpha` is the multiplier the opacity utility
sets. Multiplying (rather than replacing) means a translucent token keeps its built-in
translucency: `bg-faded bg-opacity-50` halves the _faded_ alpha rather than forcing a flat 0.5.

The three multipliers are `@property`-registered with `inherits: false` and an `initial-value` of
`1`, so the default is "unchanged" and **a value set on one element never leaks to its children**:

```css
@property --_background-alpha {
  syntax: "<number>";
  inherits: false;
  initial-value: 1;
}

:where(.bg-opacity-50) {
  --_background-alpha: 0.5;
}
```

| Variable              | Syntax     | Initial | Set by             |
| --------------------- | ---------- | ------- | ------------------ |
| `--_text-alpha`       | `<number>` | `1`     | `text-opacity-*`   |
| `--_background-alpha` | `<number>` | `1`     | `bg-opacity-*`     |
| `--_border-alpha`     | `<number>` | `1`     | `border-opacity-*` |

These sit alongside the padding/margin/radius composition variables as the non-inheriting half of
the system. See the [overview](/docs/base/utilities#composition-variables-registered-non-inheriting).
