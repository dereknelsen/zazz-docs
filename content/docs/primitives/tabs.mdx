title: Tabs

Tabs let one region show several panels, one at a time. Reach for them when content divides into peer sections — settings groups, product details, dashboard views — and you'd rather swap in place than stack everything down the page. Zazz builds tabs from native `<input type="radio">` controls, so selection, mutual exclusion, and Arrow-key navigation come from the browser with **no JavaScript**.

## Default

<ComponentPreview name="tabs-default" />

```html
<div class="tabs">
  <div class="tabs__list" role="tablist">
    <div class="tabs__indicator" aria-hidden="true"></div>

    <input type="radio" name="settings" id="tab-1" checked />
    <label for="tab-1" class="tabs__label">
      <span class="tabs__label-text">Account</span>
    </label>

    <input type="radio" name="settings" id="tab-2" />
    <label for="tab-2" class="tabs__label">
      <span class="tabs__label-text">Billing</span>
    </label>

    <input type="radio" name="settings" id="tab-3" />
    <label for="tab-3" class="tabs__label">
      <span class="tabs__label-text">Notifications</span>
    </label>
  </div>

  <div class="tabs__panel">
    <h2 class="text-h6">Account</h2>
    <p>Manage your profile, email, and password.</p>
  </div>

  <div class="tabs__panel">
    <h2 class="text-h6">Billing</h2>
    <p>Review invoices and update your payment method.</p>
  </div>

  <div class="tabs__panel">
    <h2 class="text-h6">Notifications</h2>
    <p>Choose what we email you about and how often.</p>
  </div>
</div>
```

## Anatomy

Tabs are a **segmented control**. A muted track (`.tabs__list`) holds the visually-hidden radios, their `.tabs__label`s, and a single card-colored pill (`.tabs__indicator`) that slides behind the active tab. The `.tabs__panel`s sit **after the track** as siblings, in the same order as the radios. Three mechanisms carry the component:

1. **The radios share a `name`**, so only one is ever checked and the browser handles mutual exclusion and Arrow-key movement.
2. **The pill is anchor-positioned.** The checked label exposes `anchor-name: --active-tab`; `.tabs__indicator` reads it through `anchor()` insets, so it sizes to and slides between labels. Label fills stay transparent — the pill _is_ the active fill.
3. **Panels are revealed positionally.** Because panels follow the track rather than each radio, the Nth checked radio is matched to the (N+1)th child of `.tabs` with a `:has(.tabs__list > input:nth-of-type(N):checked) > .tabs__panel:nth-child(N+1)` chain.

Two consequences of the positional reveal: **panel order must match radio order**, and the built-in chain covers up to **24 tabs** (extend the chain in `_tabs.css` if you need more).

| Part          | Element                         | Role                                                       |
| ------------- | ------------------------------- | ---------------------------------------------------------- |
| Root          | `<div class="tabs">`            | Flex column; stacks the track above the active panel       |
| Track         | `<div class="tabs__list">`      | Muted segmented track; holds radios, labels, and the pill  |
| Indicator     | `<div class="tabs__indicator">` | Card pill that slides behind the active label (decorative) |
| Trigger state | `<input type="radio">`          | Visually hidden; holds the checked state                   |
| Trigger label | `<label class="tabs__label">`   | The clickable tab inside the track                         |
| Panel         | `<div class="tabs__panel">`     | Sibling after the track; shown when its radio is checked   |

## Tokens

Every visual is a local custom property on `.tabs`, so a tab strip is retuned in one place:

| Local prop                        | Default              | Controls                              |
| --------------------------------- | -------------------- | ------------------------------------- |
| `--tabs-gap`                      | `--gap-md`           | Space between the track and the panel |
| `--tabs-track-background`         | `--muted`            | Track fill                            |
| `--tabs-track-padding`            | `--step-0_5`         | Track inset around the labels         |
| `--tabs-track-gap`                | `--step-0_5`         | Space between labels                  |
| `--tabs-track-radius`             | `--radius-md`        | Track corner radius                   |
| `--tabs-indicator-background`     | `--card`             | Sliding pill fill                     |
| `--tabs-indicator-radius`         | `--radius-sm`        | Pill corner radius                    |
| `--tabs-indicator-shadow`         | `--shadow-xs`        | Pill elevation                        |
| `--tabs-label-foreground`         | `--muted-foreground` | Resting label color                   |
| `--tabs-label-foreground--hover`  | `--foreground`       | Hovered label color                   |
| `--tabs-label-foreground--active` | `--foreground`       | Active label color                    |
| `--tabs-label-padding`            | `--step-2_5`         | Label inline padding                  |
| `--tabs-label-height`             | `--step-8`           | Label (and track) height              |

```css
/* Fill the active tab with the brand color instead of the card */
.tabs[data-variant="brand"] {
  --tabs-indicator-background: var(--primary);
  --tabs-label-foreground--active: var(--primary-foreground);
}
```

## States

- **Resting.** Label sits at `--tabs-label-foreground` (`--muted-foreground`) on the muted track, with no pill.
- **Hover.** Label color lifts to `--tabs-label-foreground--hover` (`--foreground`).
- **Active.** The checked label switches to `--tabs-label-foreground--active`, and the `.tabs__indicator` pill (`--card` + `--shadow-xs`) slides behind it; its panel is shown.
- **Focus.** The radio is visually hidden, so its `:focus-visible` ring is surfaced on the label via `input:focus-visible + label` — the standard `--ring` outline.

> **Browser support.** The sliding pill uses [CSS anchor positioning](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_anchor_positioning), which today ships in Chromium (125+) but not yet Firefox or Safari. The `anchor()` insets are gated behind `@supports`, so in browsers without it the pill is hidden and the **active label is painted directly** (`--card` fill + `--shadow-xs`). The active state stays clear everywhere — only the slide animation is lost.

## Accessibility

- Each `<label>` is tied to its radio with `for`/`id`, so clicking the label selects the tab and the hit target covers the whole label.
- Radios in one strip **must share a `name`** — that's what makes them mutually exclusive and gives free keyboard support: `Arrow` keys move between tabs and switch panels, matching the expected tab interaction.
- Mark one radio `checked` so a panel is visible on load.
- The radios are hidden with `clip-path` (not `display: none`), so they stay focusable and announced to assistive tech.
- The `.tabs__indicator` is purely decorative — keep its `aria-hidden="true"`.
- **Panel order must match radio order**, since the reveal is positional (the Nth tab shows the Nth panel after the track).
- The track carries `role="tablist"`; for richer semantics in an app you can layer `role="tab"`/`role="tabpanel"` on the labels and panels, but the visual layer doesn't depend on it.
