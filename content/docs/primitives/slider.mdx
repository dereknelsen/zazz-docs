title: Slider

A slider lets the user pick a numeric value from a continuous range: volume, opacity, font size, a price filter ceiling. Zazz styles the native `<input type="range">` element in the [reset](app/zazz/styles/_reset.css), so a plain `<input type="range">` already takes on the Zazz theme. No class and no custom JS required.

## Default

<ComponentPreview name="slider-default" />

```html
<input type="range" min="0" max="100" value="50" aria-label="Volume" />
```

The reset handles the track and thumb styling for every `input[type="range"]`. Most projects can use the bare input.

## Tokens

| Property             | Token / local                               |
| -------------------- | ------------------------------------------- |
| Track height         | `--range-track-height` (`--step-2`)         |
| Track background     | `--range-track-background` (`--muted`)      |
| Track border         | `--range-track-border`                      |
| Track radius         | `--range-track-radius` (`--radius-full`)    |
| Thumb width / height | `--range-thumb-size` (`--step-3_5`)         |
| Thumb background     | `--range-thumb-background` (`--background`) |
| Thumb border         | `--range-thumb-border`                      |
| Thumb radius         | `--range-thumb-radius` (`--radius-full`)    |
| Focus ring           | `--ring` outline (`:focus-visible`)         |

The track and thumb sizes are exposed as `--range-*` locals on the input, so a single override (e.g. `--range-thumb-size`) retunes the slider without touching the pseudo-element rules. The thumb stays centered automatically via a calc'd `--range-thumb-offset`.

The thumb is a circular knob with a primary-colored border, sitting on a muted track. The active region (left of the thumb) is the same color as the inactive region; sliders don't fill the track by default (kept consistent across rendering engines). Add a custom paint via JavaScript and CSS if you need a filled track.

## States

- **Default.** Thumb at the current value, track muted.
- **Focus.** The standard `:focus-visible` outline ring applies (`outline-color: var(--ring)`), consistent with every other control — no extra wiring needed.
- **Disabled.** Set `disabled` on the input. The track and thumb mute further.

## Form composition

```html
<div class="field">
  <label class="field__label" for="volume">Volume</label>
  <input type="range" id="volume" min="0" max="100" value="50" />
  <output class="field__hint" for="volume">50</output>
</div>
```

The `<output>` element pairs with the slider to display the current value. Wire it with a small JS listener:

```js
const slider = document.querySelector("#volume");
const output = document.querySelector('output[for="volume"]');
slider.addEventListener("input", () => (output.value = slider.value));
```

## Accessibility

- The native `<input type="range">` handles keyboard navigation (`Arrow` keys, `Home`/`End`) and screen reader announcement automatically.
- Always provide a `<label>` (or `aria-label`) describing what the slider controls.
- For values that aren't immediately meaningful as numbers (a temperature in degrees, a percentage), pair with `<output>` so the value is visible to sighted users in real time.
- Use `min`, `max`, and `step` to define the valid range. The browser communicates these to assistive tech via the implicit `aria-valuemin`, `aria-valuemax`, and `aria-valuenow` properties.
