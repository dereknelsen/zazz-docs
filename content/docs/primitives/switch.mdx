---
title: Switch
description: Visual toggle for binary settings. Built on the checkbox semantic.
---

A switch is the right control when a setting takes effect immediately on toggle: turning a feature on, switching notifications, enabling autoplay. For choices that apply on submit (terms acceptance, multi-select filters), use [Checkbox](/docs/primitives/checkbox) instead. Both controls produce the same value semantically; the choice is one of UX expectation.

## Default

<ComponentPreview name="switch-default" />

```html
<label class="field">
  <input type="checkbox" role="switch" />
  <span class="field__label">Enable notifications</span>
</label>
```

The switch is a native `<input type="checkbox">` with `role="switch"` — so it behaves like a checkbox semantically and submits as a form value, but announces as a switch. The reset (`appearance: none`) redraws it as a pill track with a sliding thumb. **No class is needed on the input.** Wrap it in a `<label class="field" >` to sit the control beside its `.field__label`.

## Tokens

| Property               | Token / value                       |
| ---------------------- | ----------------------------------- |
| Track width            | `--step-9`                          |
| Track height           | `--step-5`                          |
| Track background (off) | `--muted`                           |
| Track background (on)  | `--primary`                         |
| Thumb size             | `--step-5`                          |
| Thumb background       | `--white`                           |
| Track radius           | `--radius-full`                     |
| Inline gap             | `--step-2` (`.field[]`)             |
| Focus ring             | `--ring` outline (`:focus-visible`) |

The thumb is a `::after` pseudo on the wrapping `<label>`, positioned over the track and slid with `translate` when the input is `:checked`. Pseudo-elements on replaced inputs are unreliable, so the label carries the knob.

## States

- **Off.** Track is `--muted`, thumb sits at the left.
- **On.** Track becomes `--primary`, thumb slides to the right (`:checked`).
- **Focus.** The focus ring is a `:focus-visible` outline (`outline-color: var(--ring)`).
- **Disabled.** Set `disabled` on the input. The shared disabled state applies `opacity: 0.5` and `cursor: not-allowed`.

## Composing in a form

```html
<div class="flex flex-col gap-xs">
  <label class="field">
    <input type="checkbox" role="switch" />
    <span class="field__label">Email notifications</span>
  </label>
  <label class="field">
    <input type="checkbox" role="switch" checked />
    <span class="field__label">Two-factor authentication</span>
  </label>
</div>
```

Each switch is its own `.field` row; stack them in a flex column for a settings list.

## Accessibility

- `role="switch"` makes screen readers announce the control as a switch (on/off) rather than a checkbox (checked/unchecked). It remains a native checkbox underneath, so keyboard interaction (Space toggles) and form submission are unchanged.
- The label must convey what the switch controls. Bad: "Notifications." Better: "Enable email notifications." The user should know what happens when they flip it.
- Switches that take effect immediately don't need a save button. If the switch sits inside a form that requires submission, make that clear in the form's helper text.
